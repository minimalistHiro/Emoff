# Create Markdown Skill

## 概要

ユーザーからマークダウンファイルの作成依頼が来たときに、**ファイル名をすべて大文字**にしてファイルを作成する。

## ルール

### ファイル名は必ず大文字にする

- 英字はすべて大文字（例: `my_document.md` → `MY_DOCUMENT.md`）
- 単語の区切りはアンダースコア `_` を使用する
- 拡張子は `.md`（小文字）
- 例:
  - `release_note.md` → `RELEASE_NOTE.md`
  - `setup_guide.md` → `SETUP_GUIDE.md`

## 手順

### 1. 作成するファイルの内容・場所を確認する

依頼が曖昧な場合は以下を確認する：

- **ファイル名・テーマ**: 何に関するドキュメントか
- **保存先**: どのディレクトリに保存するか（指定がなければプロジェクトルートを使用）
- **内容**: 記載すべき内容の概要

ユーザーがある程度の情報を伝えている場合は確認を最小限にし、すぐに作成に進む。

### 2. ファイル名を決定する（大文字に変換）

- ユーザーが指定したファイル名をすべて大文字に変換する
- 英字以外（日本語・数字・記号）はそのまま使用する
- スペースはアンダースコア `_` に変換する
- 拡張子は `.md` を付ける

**変換例:**

| ユーザーの指定 | 作成するファイル名 |
|---|---|
| `release_note` | `RELEASE_NOTE.md` |
| `setup guide` | `SETUP_GUIDE.md` |
| `readme` | `README.md` |
| `my document` | `MY_DOCUMENT.md` |

### 3. ファイルを作成する

Write ツールを使って指定のパスにファイルを作成する。

- 保存先の指定がない場合はプロジェクトルート `/Users/kanekohiroki/Desktop/emoff/` に保存する
- ユーザーから受け取った情報でファイルの内容を記載する
- 内容の指定がない場合は、テーマに合った基本的なテンプレートを作成する

### 4. 作成結果を報告する

作成したファイルのフルパスとファイル名を日本語で報告する。

## 対象となる依頼例

- マークダウンを作成して
- MDファイルを作って
- マークダウンファイルを作成して
- `release_note` というマークダウンを作って
- `setup_guide.md` を作成して
- 〇〇に関するマークダウンを作って

## 注意事項

- **ファイル名は必ずすべて大文字にする。これは最重要ルールである。**
- ユーザーが小文字のファイル名を指定していても、自動的に大文字に変換して作成する。
- ユーザーへの確認なく大文字に変換してよい（仕様として周知済み）。
- 既存ファイルを上書きしないよう、作成前に Glob ツールで同名ファイルが存在しないか確認する。
