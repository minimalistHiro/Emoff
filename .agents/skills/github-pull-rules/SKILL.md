# GitHub Pull Rules

## 概要

GitHubのリモートリポジトリから最新の `main` ブランチをローカルに取り込む。
必ず `git fetch` でリモート情報を最新化してからプルを行うこと。

## 手順

### Step 1: ローカルの未コミット変更を確認する

```bash
git status --short
```

未コミットの変更がある場合は、プル前にユーザーに以下のいずれかを確認する：
- 先にコミットしてからプルするか
- 一時的にスタッシュしてプル後に戻すか
- 変更を破棄してプルするか（破棄は明示承認後のみ）

### Step 2: リモート情報をフェッチする

```bash
git fetch --all
```

フェッチ後、`origin/main` の最新コミット時刻とメッセージを表示してユーザーに確認させる。

```bash
git log origin/main -1 --format="%ci %s"
```

### Step 3: 実行手順をユーザーに提示する

以下の手順を日本語でユーザーに提示し、実行の承認を得る：

1. `main` ブランチに切り替え（`git checkout main`）
2. リモートの最新を取り込む（`git pull origin main`）

すでに `main` ブランチにいる場合は Step 3-1 はスキップする旨を明記する。

### Step 4: ユーザーの承認後、実行する

```bash
git checkout main
git pull origin main
```

### Step 5: 結果を報告する

取り込んだコミット数と最新コミットのメッセージ・時刻を日本語で報告する。

## 対象となる依頼例

- GitHubからプルして
- 最新をプルして
- リモートからプルして
- mainをプルして
- 最新の状態にして
- GitHubの最新をローカルに反映して
- リモートの変更を取り込んで

## 注意事項

- 必ず `git fetch --all` を最初に実行し、ローカルのリモート追跡情報を最新化すること。
- 実行手順をユーザーに提示してから承認を得て実行すること。
- ローカルに未コミットの変更がある場合は、プル前にユーザーに確認すること。**ユーザーの明示承認なく `git stash` / `git checkout --` 等の破壊的操作は行わない。**
- 作業ブランチで作業中だった場合、main に戻すと作業内容を見失いやすいため、ユーザーに「main に戻すと現在ブランチの作業が一旦保留になる」旨を伝えてから実行する。
