# TODO Add

## 概要

TODO項目を追加する際に、TODO.md と COMPLETED.md の両方を読み取って使用済みIDを把握し、重複しないIDを採番した上で項目を追加する。

## 手順

### 1. 両ファイルの読み込みとID収集

以下の2ファイルを Read ツールで読み込む：

- `/Users/kanekohiroki/Desktop/emoff/TODO.md`
- `/Users/kanekohiroki/Desktop/emoff/COMPLETED.md`

両ファイルから、各プレフィックス（現状は `A-` のみ）ごとに **使用済みの全IDを収集** する。

### 2. 次のIDを採番

追加する項目が属するセクションのプレフィックスを特定し、そのプレフィックスの使用済み最大番号 + 1 を新IDとする。

**現行のプレフィックス運用：**
- `A-xx`：emoff の全タスクで使用中（機能実装・バグ修正・改善等を区別せず一本化）

将来プレフィックスを追加する場合は、本スキルの「セクション判定基準」にも追記すること。

**採番例：**
- TODO.md に A-25〜A-26、COMPLETED.md に A-1〜A-24, A-27 がある場合 → 次は **A-28**

### 3. 項目の内容を確定

ユーザーの依頼内容から以下を決定する：

- **優先度**：最高 / 高 / 中 / 低（長期）
- **内容**：実装・対応内容の概要（必要に応じて太字でタイトル化）
- **状態**：`[ ] 未着手`（デフォルト）

ユーザーが優先度を明示していない場合は、内容から判断して提案する。

### 4. TODO.md に追記

「## タスク一覧」のテーブル末尾に新しい行を Edit ツールで追記する。

**フォーマット：**
```
| {ID} | {優先度} | {内容} | [ ] 未着手 |
```

既存テーブル（`| ID | 優先度 | 内容 | 状態 |`）の末尾行として挿入する。ヘッダー行・区切り行は変更しない。

### 5. 複数項目の一括追加

複数項目を同時に追加する場合は、各項目について手順2〜4を繰り返す。IDは連番で採番する（A-28, A-29, A-30 ...）。

### 6. 結果報告

追加した項目のID・優先度・内容を日本語で報告する。

## 対象となる依頼例

- TODOに追加して
- タスクを追加して
- バグを追加して
- 新しいTODOを追加して
- これをTODOに入れて
- 〇〇をTODOに加えて
- 以下をTODOリストに含めて
- まずTODOに追加してください

## 注意事項

- **IDの採番時は必ず TODO.md と COMPLETED.md の両方を確認すること。** これを怠るとID重複が発生する。
- IDの再利用は禁止。廃止された項目のIDも再利用してはならない。
- 既存の項目と内容が重複していないか、追加前にTODO.mdの既存項目を確認する。重複の可能性がある場合はユーザーに確認する。
- TODO.md のルール（IDの再利用禁止・状態形式）はファイル冒頭の「ルール」セクションに従うこと。
