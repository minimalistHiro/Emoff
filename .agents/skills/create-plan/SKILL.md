# Create Plan Skill

## 概要

ユーザーから計画書・プランの作成依頼が来たときに、`/Users/kanekohiroki/Desktop/emoff/plans/in_progress/` にマークダウン形式の計画書ファイルを作成する。新規作成の計画書は未完了のため、必ず `in_progress/` フォルダに配置する。

## 手順

### 1. 計画書の内容をユーザーに確認する

依頼が曖昧な場合は、以下を確認する：

- **プラン名・テーマ**: 何に関する計画か（例：「機能Xの実装計画」「v1.0リリース計画」）
- **概要**: 計画の目的・背景
- **ゴール**: 達成したいこと
- **タスク・手順**: 具体的なステップ
- **スケジュール**: 目標日程（あれば）

ただし、ユーザーがある程度の内容を伝えている場合は確認を最小限にし、すぐに作成に進む。

### 2. ファイル名を決定する

以下のルールでファイル名を決める：

- 形式: `YYYYMMDD_<プラン名をスネークケースで>.md`
- 例: `20260401_chat_feature_plan.md`
- 日付は今日の日付（`currentDate` コンテキストから取得）を使用する

### 3. 計画書ファイルを作成する

`/Users/kanekohiroki/Desktop/emoff/plans/in_progress/<ファイル名>` に以下のテンプレートで Write ツールを使って作成する。

```markdown
# <プラン名>

- **作成日**: YYYY-MM-DD
- **ステータス**: 検討中 / 承認済み / 進行中 / 完了 / 中止
- **優先度**: 高 / 中 / 低
- **対象バージョン**: vX.X.X（該当する場合）

---

## 概要

（プランの目的・背景を記載）

## ゴール

（達成したいことを箇条書きで記載）

## タスク・手順

- [ ] タスク1
- [ ] タスク2
- [ ] タスク3

## スケジュール

| マイルストーン | 目標日 |
|---|---|
| 〇〇 | YYYY-MM-DD |

## メモ・備考

（補足事項があれば記載）
```

ユーザーから受け取った情報でテンプレートの各項目を埋めること。情報がない項目はテンプレートの文言をそのまま残すか、空欄にする。

### 4. 作成結果を報告する

作成したファイルのパスとプラン名を日本語で報告する。

## 対象となる依頼例

- プランを作成して
- 計画書を作成して
- 〇〇の計画書を作って
- 〇〇のプランを書いて
- 〇〇の実装計画を立てて
- 〇〇のロードマップを作成して
- 計画を立ててほしい

## 注意事項

- ファイルの保存先は必ず `/Users/kanekohiroki/Desktop/emoff/plans/in_progress/` とする。
- ファイル名は日付から始めることで、時系列で並び替えできるようにする。
- 既存の計画書を更新する場合は Write ではなく Edit ツールを使う。
- 完了済みの計画書は `plans/completed/` に、未完了の計画書は `plans/in_progress/` に配置する。
