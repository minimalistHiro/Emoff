# QA Checklist Check Skill

## 概要

アプリの現在のバージョンに対応するQAチェックリストを読み込み、内容を表示・回答する。

QAチェックリストは「探索テスト」方式を採用しており、APP_STORE_TEXTS.md の更新情報や完了TODO項目に基づく主要変更点のみをチェック項目としている。項目数は最小限（10〜15項目程度）に抑えられている。

## 手順

### 1. アプリバージョンの取得

`Read` ツールで `/Users/kanekohiroki/Desktop/emoff/pubspec.yaml` を読み込み、`version:` 行からバージョン番号を取得する。

- 例：`version: 1.0.0+1` → バージョンは `1.0.0`
- `+` 以降のビルド番号は除外する

### 2. QAチェックリストの読み込み

取得したバージョン番号を使い、以下のパスのファイルを `Read` ツールで読み込む：

```
/Users/kanekohiroki/Desktop/emoff/qa/QA_V{バージョン}.md
```

### 3. ファイル・ディレクトリが存在しない場合

- **`qa/` ディレクトリ自体が存在しない場合**: emoff では QA 運用がまだ整備されていない旨を案内する。最初のチェックリストを作成するには、`todo-complete` スキル経由で完了TODOからチェック項目を追加するか、`mkdir -p qa` でディレクトリを作成して空のチェックリストを設置することを提案する。
- **ディレクトリは存在するが該当バージョンのファイルが無い場合**: `qa/` ディレクトリ内の既存ファイル一覧を表示し、「現在のバージョン V{バージョン} に対応するQAチェックリストが見つかりません。既存のチェックリストは以下です：」と案内する。

### 4. 内容の表示・回答

チェックリストの内容を読み込んだ後、以下の情報を整理して報告する：

- **対象バージョン**: 読み込んだバージョン番号
- **進捗状況**: `- [x]`（完了）と `- [ ]`（未完了）の件数を集計し、進捗率を表示
- **チェック項目一覧**: 全項目を表示（項目数が少ないためカテゴリ分けは不要）
- **自由記述欄**: 自由記述欄に記入がある場合はその内容も表示
- **QA_RESULTS.md との関係**: テスト結果（Pass / Fail / Skip）が `QA_RESULTS.md` に記録されている場合は、その記録の存在を併記する（詳細は別途 `QA_RESULTS.md` 参照と案内）

## 対象となる依頼例

- チェックリストを見て
- チェックリストを確認して
- QAチェックリストを見せて
- テスト項目を確認して
- チェックリストの進捗は？
- 今バージョンのQAは？
- どこまでテストが終わってる？

## 注意事項

- バージョン番号は必ず `pubspec.yaml` から動的に取得すること。ハードコードしない。
- ビルド番号（`+` 以降）は除外してファイル名を構成すること。
- チェックリストの内容を改変せず、そのまま表示すること。
- テスト結果の更新（Pass/Fail への変更）は本スキルでは行わない。それは `QA_RESULTS.md` への追記運用で行う。
- emoff にはスプレッドシート連携が存在しないため、本スキルはローカルマークダウンのみを参照する。
