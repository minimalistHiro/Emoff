# 画面詳細設計 進捗管理

> 最終更新: 2026-04-03

---

## このファイルの目的

APP_SCREENS.md に各画面の詳細UI設計を段階的に追加していくための進捗管理ファイル。
新しいスレッドを開始するたびにこのファイルを読み込み、「次はどこから進めるか」を判断する起点とする。

---

## 運用フロー

```
1. 新しいスレッドを開始する
2. 「設計を進めて」または「次の設計は？」と依頼する
   → design-navigator スキルが起動し、このファイルと関連ドキュメントを読み込む
3. 次に設計すべき画面と、事前に確認・決定すべき事項の案内を受ける
4. 1画面ずつ詳細設計を書き、レビュー・修正を経て確定する
5. 確定後、plans/in_progress/ に設計書ファイルを作成し、APP_SCREENS.md にリンクを追記、このファイルの該当行を完了に更新する
6. 実装が完了したら、plans/in_progress/ → plans/completed/ に設計書を移動する
```

---

## 参照ドキュメント

設計時に毎回チェックするドキュメント一覧。

| ドキュメント | 役割 | いつ参照するか |
|---|---|---|
| `SCREEN_DESIGN.md` | 画面概要・遷移図 | 設計対象画面の概要と遷移先の確認 |
| `FIRESTORE.md` | データベース構造 | 表示・操作するデータのフィールド確認 |
| `BUSINESS_MODEL.md` | 仕様・プラン制約 | Free/Pro/Businessの機能制限、AI変換仕様 |
| `UI_UX.md` | デザインルール・共通コンポーネント | 色・フォント・ウィジェットのルール準拠 |
| `APP_SCREENS.md` | 画面一覧・遷移図・設計書リンク | 画面構成の全体像確認、設計書ファイルへのリンク |
| `plans/in_progress/` | 設計書（未実装） | 設計済み・未実装の詳細設計書の格納先 |
| `plans/completed/` | 設計書（実装済み） | 実装完了した設計書の格納先 |
| `REVIEW_ITEMS.md` | 未決定事項 | ブロッカーの有無確認 |
