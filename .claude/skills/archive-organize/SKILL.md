# Archive Organize Skill

## 概要

要検討事項（REVIEW_ITEMS.md）の項目をアーカイブ（ARCHIVED_IDEAS.md）に移動する対話的なワークフロー。ユーザーが移動対象と理由を指示した後、両ファイルを一括で整理する。

## 手順

### 1. 現状表示

`/Users/kanekohiroki/Desktop/emoff/REVIEW_ITEMS.md` を Read ツールで読み取り、全項目（R-xx）を一覧で表示する。ユーザーに「どの項目をアーカイブに移行するか」を確認する。

### 2. ユーザーからの指示を受ける

ユーザーが以下を伝える：
- アーカイブ対象のID（例：R-8, R-11）
- 各項目または項目グループのアーカイブ理由（例：「方針転換により不要」「却下と決定」「凍結（後フェーズで再検討）」）

指示内容をまとめてユーザーに最終確認する。

### 3. マークダウン更新

確認が取れたら以下を実行する：

1. **REVIEW_ITEMS.md** から指定された項目のセクション（`### R-x: タイトル` 〜 次の `---` までのブロック）を Edit ツールで削除する。
2. **ARCHIVED_IDEAS.md** を Read ツールで読み込み、「アーカイブ一覧」のテーブル末尾に行を追記する：
   - **ID**: 元のIDをそのまま保持（再採番禁止）
   - **カテゴリ**: REVIEW_ITEMS 側の「カテゴリ」をそのまま転記
   - **内容**: REVIEW_ITEMS 側の「内容」を1行に要約して転記（改行・複数行説明は1行にまとめる）
   - **元の優先度**: REVIEW_ITEMS 側の優先度をそのまま転記
   - **アーカイブ理由**: ユーザーが伝えた理由
   - **アーカイブ日**: 当日の日付（YYYY-MM-DD）
3. テーブル全体を **ID番号順（R-xx の数字昇順）** に並び替える。

### 4. 結果報告

移動した項目数・ID・アーカイブ理由をまとめて日本語で報告する。

## 対象となる依頼例

- アーカイブを整理したい
- アーカイブに移動したい
- 要検討事項をアーカイブして
- R-8をアーカイブに移動して
- 要検討事項を整理してアーカイブしたい
- アーカイブに入れたい項目がある

## 注意事項

- マークダウン更新前に必ずユーザーの確認を取ること。
- ARCHIVED_IDEAS.md のテーブルは常に **ID番号順** を維持すること。
- 元のID（R-xx）はそのまま保持し、採番し直さないこと。REVIEW_ITEMS.md 側でも R-xx を欠番のままにする（再利用禁止）。
- emoff にはスプレッドシート連携がないため、本スキルではスプレッドシート操作は一切行わない。
- 将来 ARCHIVED_IDEAS.md から REVIEW_ITEMS.md に項目を戻す場合は、本スキルではなく手動操作（または別スキル）で対応する。
