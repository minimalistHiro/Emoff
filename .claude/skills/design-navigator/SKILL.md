# Design Navigator

## 概要

画面の詳細設計を1画面ずつ段階的に進めるためのナビゲーションスキル。
進捗管理ファイル（DESIGN_PROGRESS.md）を起点に、次に設計すべき画面の特定・関連情報の読み込み・設計作業のガイドを行う。

## 手順

### ステップ1: 現状把握（必ず最初に実行）

以下のファイルを Read ツールで読み込む:

1. `/Users/kanekohiroki/Desktop/emoff/DESIGN_PROGRESS.md` — 進捗状態の確認
2. `/Users/kanekohiroki/Desktop/emoff/REVIEW_ITEMS.md` — 未決定事項・ブロッカーの確認

### ステップ2: 次の設計対象を特定

DESIGN_PROGRESS.md の設計進捗テーブルから、以下の優先順で次の設計対象を特定する:

1. `[ ] 進行中` の画面があれば、それを最優先で継続
2. なければ、現在のフェーズ内で `[ ] 未着手` かつブロッカーなしの画面を順番に選択
3. フェーズ内の全画面が完了していれば、次のフェーズに進む
4. `[!] ブロック中` の画面がある場合、ブロッカーの解決を先に提案する

### ステップ3: 設計対象画面の関連情報を読み込む

次の設計対象が決まったら、以下のファイルを Read ツールで読み込む:

1. `/Users/kanekohiroki/Desktop/emoff/SCREEN_DESIGN.md` — 対象画面の概要・遷移先の確認
2. `/Users/kanekohiroki/Desktop/emoff/FIRESTORE.md` — 対象画面で使用するデータ構造
3. `/Users/kanekohiroki/Desktop/emoff/BUSINESS_MODEL.md` — 仕様・制約の確認
4. `/Users/kanekohiroki/Desktop/emoff/UI_UX.md` — デザインルール
5. `/Users/kanekohiroki/Desktop/emoff/APP_SCREENS.md` — 既存の詳細設計との一貫性確認

### ステップ4: ユーザーに報告・提案

以下の形式で報告する:

```
## 現在の進捗
- フェーズX: Y/Z 完了
- 全体: A/14 完了

## 次の設計対象
**画面名**: 〇〇画面
**フェーズ**: X（〇〇フロー）

## 事前確認事項
- （ブロッカーがあれば記載。なければ「なし」）
- （未決定事項で先に決めるべきことがあれば記載）

## 設計に必要な情報（読み込み済み）
- SCREEN_DESIGN.md: 〇〇画面の概要は「...」
- FIRESTORE.md: 使用するコレクションは「...」
- APP_SCREENS.md: 関連する既存画面は「...」

## 進め方
このまま設計を開始しますか？
```

### ステップ5: 設計の実行

ユーザーから承認を得たら:

1. APP_SCREENS.md の既存画面（特に同フェーズの画面）のフォーマット・粒度を踏襲して詳細設計を作成する
2. 以下のセクションを必ず含める:
   - 画面概要
   - 画面構成（上から順に各UI要素の詳細テーブル）
   - インタラクション（アクションと動作のテーブル）
   - デザインノート
3. 色コード・フォント・コンポーネント名はUI_UX.mdのルールに準拠する
4. Firestoreのフィールド名と対応するUI要素の関係を明記する

### ステップ6: 確定・更新

ユーザーのレビュー後、確定したら:

1. APP_SCREENS.md に詳細設計を追記する（セクション番号は既存の続番）
2. DESIGN_PROGRESS.md の該当行の状態を `[x] 完了（日付）` に更新する
3. DESIGN_PROGRESS.md のサマリーテーブルの数値を更新する
4. DESIGN_PROGRESS.md の設計ログに記録を追加する

## 対象となる依頼例

- 設計を進めて
- 次の設計は？
- 設計の進捗は？
- どこまで進んだ？
- 次は何をすべき？
- 〇〇画面の詳細設計を書いて
- 設計の状況を確認して
- 次の画面に進もう

## 注意事項

- 1回のスレッドで扱う画面は原則1〜2画面とする（コンテキスト品質の維持）
- ブロッカーがある画面は、ブロッカーの解決（REVIEW_ITEMS.mdの該当項目の決定）を先に行う
- 設計中に新たな未決定事項が出た場合は、REVIEW_ITEMS.md に追加する
- 「進行中」の画面がスレッドをまたぐ場合は、DESIGN_PROGRESS.md に作業メモを残す
