# QA テスト結果

動画・手動確認・CSV反映などで確定したテスト結果を、このファイルに記録する。

---

## 記載フォーマット

### チェック項目の結果

```
- [x] `Vxxx-NNN` **項目名** — 確認内容の説明 ✅ Pass | 確認者: 〇〇 (1人目) | 確認日時: YYYY/MM/DD | 証跡: <動画URL / スクショパス / 備考>
- [ ] `Vxxx-NNN` **項目名** — 確認内容の説明 ❌ Fail | 確認者: 〇〇 (1人目) | 確認日時: YYYY/MM/DD | 証跡: <動画URL / スクショパス / 備考>
```

- ステータス: `✅ Pass` / `❌ Fail` / `⏭ Skip`
- 確認者欄には1人目／2人目を明記する（ダブルチェック方式）。
- 1人目・2人目の両方が Pass した時点で `qa/QA_Vx.x.x.md` 側のチェックボックスを完了扱いにする。

## 運用ルール

- チェックボックスの更新（進捗管理）は、各バージョンの `qa/QA_Vx.x.x.md` 側で行う。
- このファイルには Pass / Fail / Skip のように **結果が確定した項目のみ** を記録する。
- 操作中に気づいた点・不具合・改善案は、TODO.md や REVIEW_ITEMS.md に直接追記する（このファイルには書かない）。
- バージョンごとに見出し（`## vX.X.X`）で区切って記載する。
