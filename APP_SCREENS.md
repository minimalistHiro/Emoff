## 1. ホーム画面（友達一覧） — Circle of Clarity

### 画面概要

友達登録済みのユーザー一覧を表示するメイン画面。
「Circle of Clarity（明晰さの輪）」というコンセプトで、感情を排した客観的なコミュニケーションを行う信頼できる仲間のリストを提供する。

### 画面構成（上から順）

#### 1. AppBar（固定ヘッダー）

| 要素 | 位置 | 詳細 |
|------|------|------|
| ハンバーガーメニューアイコン | 左端 | `menu` アイコン。サイドメニュー（Drawer）を開く |
| 画面タイトル「FRIENDS」 | 左寄せ（アイコン右） | 大文字・セミボールド・トラッキング広め |
| 友達追加アイコン | 右側① | `person_add` アイコン。友達申請画面へ遷移 |
| 検索アイコン | 右側② | `search` アイコン。検索モードを展開 |

- **実装時**: `CustomAppBar` を使用（UI_UX.mdルール準拠）
- **背景色**: `#0D0D0D`（ダークテーマ）
- **アイコン色**: `#00D4FF`（シアン）

#### 2. ヒーローセクション

| 要素 | 詳細 |
|------|------|
| メインタイトル | 「Circle of Clarity」— 極太（ExtraBold）、36〜40px、フォント: Manrope系ヘッドライン |
| サブテキスト | 「Your workspace for objective communication with trusted peers. Quality over quantity.」— 通常ウェイト、14〜16px、テキスト色（副） |

- **配置**: 左寄せ、AppBar下に十分な余白（24px以上）
- **テキスト色**: タイトル → `#FFFFFF`、サブテキスト → `#A0A0A0`
- **下マージン**: 48px（リスト領域との十分な間隔）

#### 3. 検索バー

| 要素 | 詳細 |
|------|------|
| フィルターアイコン | 左側に `filter_list` アイコン |
| プレースホルダー | 「Search circle...」 |
| 形状 | 角丸フル（カプセル型 / StadiumBorder） |
| 背景色 | `#1A1A1A`（カード背景色） |

- **実装時**: `CustomTextField` をベースにカスタマイズ
- **下マージン**: 32px

#### 4. 友達リスト（カード形式）

各友達は横並びのカード（ListTile風）で表示。

**カード1件あたりの構成:**

| 要素 | 詳細 |
|------|------|
| アバター画像 | 56×56px、円形。デフォルトではグレースケール表示、タップ/ホバー時にカラー化 |
| オンラインステータス | アバター右下に小さな丸（直径14px）。オンライン=シアン（`#00D4FF`）、オフライン=非表示または灰色 |
| 名前 | ボールド、16px、`#FFFFFF` |
| 肩書き/ステータステキスト | 大文字、12px、レタースペーシング広め、`#A0A0A0` |
| 右矢印アイコン | `chevron_right`、`#555555`、タップで友達プロフィール画面へ遷移 |

**カードスタイル:**
- 背景色: `#1A1A1A`（surface-container-low相当）
- 角丸: 12〜16px（xl相当）
- パディング: 上下左右20px
- カード間マージン: 12〜16px
- タップ時背景色: `#242424`（elevate相当）

#### 5. 招待セクション（Empty State / CTA）

友達リスト下部に表示される、友達招待を促すエリア。

| 要素 | 詳細 |
|------|------|
| 枠線 | 破線ボーダー（dashed）、角丸フル（rounded-full / 大きな角丸） |
| プラスアイコン | `add_circle`、36px、`#555555` |
| テキスト | 「Expand your objective circle」— セミボールド、`#A0A0A0` |
| 招待ボタン | 「Invite Friend」— `CustomButton`（primary バリアント） |

- **配置**: 中央寄せ、パディング48px
- **背景**: 半透明のsurface色

#### 6. FAB（フローティングアクションボタン）

| 要素 | 詳細 |
|------|------|
| アイコン | `add`（プラス） |
| サイズ | 56×56px |
| 位置 | 右下固定（ボトムナビの上、right: 24px, bottom: 112px相当） |
| 背景色 | `#00D4FF`（シアン） |
| アイコン色 | `#0D0D0D`（黒） |

- **機能**: 友達申請画面へ遷移（AppBarの `person_add` と同機能のショートカット）
- **表示条件**: モバイルのみ表示

#### 7. ボトムナビゲーション（共通コンポーネント）

| タブ | アイコン | ラベル | 状態 |
|------|----------|--------|------|
| 友達 | `group` | FRIENDS | **アクティブ**（アイコンFILL=1、背景ハイライト） |
| チャット | `chat_bubble` | CHATS | 非アクティブ |
| 設定 | `settings` | SETTINGS | 非アクティブ |

**スタイル:**
- 背景: `#0D0D0D` + ブラー効果（backdrop-blur）
- アクティブタブ: シアンアイコン + `#1A1A1A`背景の角丸チップ
- 非アクティブ: `#555555`
- ラベル: 大文字、11px、レタースペーシング広め
- 上部ボーダー: 極細（1px）、`#242424`

### インタラクション

| アクション | 動作 |
|------------|------|
| 友達カードをタップ | 友達プロフィール画面へ遷移 |
| `person_add` アイコンをタップ | 友達申請画面へ遷移 |
| `search` アイコンをタップ | 検索モード展開（AppBarが検索フィールドに変化） |
| 検索バーに入力 | 友達リストをリアルタイムフィルタリング |
| FAB（+）をタップ | 友達申請画面へ遷移 |
| 「Invite Friend」ボタンをタップ | 友達招待フロー（共有シート等） |
| ハンバーガーメニューをタップ | サイドドロワーを開く |

### デザインノート

- **グレースケール写真**: アバターは通常グレースケールで表示し、ホバー/タップでカラーに変化するアニメーション（duration: 500ms）。「感情を排す」コンセプトを視覚的に表現
- **タイポグラフィ**: ヘッドライン系はManrope（太め）、ボディ/ラベル系はInter（標準）の二書体構成
- **レタースペーシング**: 肩書き等の小テキストは `tracking-widest`（大文字+広めのスペーシング）でエディトリアルな印象
- **レイアウト**: モバイルは1カラム、タブレット以上は2カラムグリッド
- **ソート**: デスクトップでは「Recent」「A-Z」のフィルターボタンを表示（モバイルでは非表示）

## 2. ログイン画面 — Welcome Back

### 画面概要

メールアドレスとパスワードでログインする画面。
「Welcome Back — Access your workspace for clarity.」というコピーで、Emoffの哲学（感情を排した明晰なコミュニケーション）を訴求する。

### レイアウト構成

**モバイル**: フォームカードのみ（中央配置、縦スクロール不要）
**デスクトップ（タブレット以上）**: 左右2カラム（左: ブランディングビジュアル、右: ログインフォーム）

※ Flutterモバイルアプリとしてはモバイルレイアウトのみ実装

| 配置 | 中央寄せ |

- **テキスト色**: `#00D4FF`（シアン）
- **下マージン**: 48px

#### 2. ウェルカムヘッダー

- **配置**: 左寄せ
- **タイトル色**: `#FFFFFF`

- **下マージン**: 40px

#### 3. メールアドレス入力フィールド

| 入力フィールド | プレースホルダー「name@domain.com」 |

#### 4. パスワード入力フィールド

| 要素 | 詳細 |
|------|------|
| メイン背景 | `#0D0D0D`（ダーク） |

| アンビエントグロー（任意） | 画面右上と左下に、シアン/パープルの大きなぼかし円（blur: 120px、opacity: 5%）を配置し、奥行き感を演出 |

### インタラクション

| アクション | 動作 |
|------------|------|
| ログインボタンをタップ | Firebase Auth でメール+パスワード認証 → 成功時ホーム画面へ遷移 |
| 「FORGOT PASSWORD?」をタップ | パスワードリセット画面へ遷移 |
| 「Sign Up」をタップ | 新規登録画面へ遷移 |
| バリデーションエラー | `CustomDialog` でエラーメッセージ表示（スナックバー禁止） |
| 認証エラー | `toUserFriendlyError()` で変換した日本語メッセージを `CustomDialog` で表示 |

### デザインノート

- **ミニマル構成**: フォーム要素のみの潔い画面。不要な装飾を排除し、Emoffの「感情を排す」哲学を体現
- **ラベルスタイル**: すべて大文字+広めのレタースペーシング（tracking-widest）で、エディトリアル/ハイエンドな印象
- **フォームカード**: モバイルでは画面中央に浮かぶカード風のレイアウト（角丸大・軽い影）
- **デスクトップ左カラム（参考）**: 「Objective Communication.」のキャッチコピーとミニマルな建築写真を配置。モバイルアプリでは不要だが、Web版展開時の参考として保持

## 3. トークルーム画面 — AI Neutralization Chat

### 画面概要

個別チャット画面。Emoffの核心機能である「AIトーン中立化（Neutralization）」がここで動作する。
ユーザーが入力したメッセージをAIが感情・スラング・絵文字を除去して客観的なトーンに変換し、変換前後のプレビューを表示した上で送信する。

### 画面構成（上から順）

#### 1. AppBar（固定ヘッダー）

| 要素 | 位置 | 詳細 |
|------|------|------|
| 戻るボタン | 左端 | `arrow_back` アイコン。トーク一覧画面に戻る |
| 相手の名前 | 左寄せ（アイコン右） | セミボールド、18px、Manrope |
| 相手の肩書き | 名前の下 | 大文字、10px、レタースペーシング広め（tracking-widest）、`#555555` |
| 検索アイコン | 右側① | `search` アイコン。トーク内検索 |
| メニューアイコン | 右側② | `more_vert` アイコン。オプションメニュー |

- **実装時**: `CustomAppBar` を使用
- **背景色**: `#0D0D0D`
- **下部**: 1pxの区切り線（`#242424`）

#### 2. メッセージキャンバス（スクロール領域）

メッセージの表示エリア。上部に余白、下部はインプットドックの高さ分の余白（160px相当）を確保。

| 背景色 | `#1A1A1A`（surface-container-low相当） |

##### 受信メッセージ（相手のバブル）

##### 「NEUTRALIZED」バッジ

AI変換済みメッセージに付与されるラベル。

#### 3. AI変換プレビュートレイ（ボトムシート風）

**Emoffの最重要UIコンポーネント**。「確認」ボタンタップ時に画面下部からスライドアップで表示される。

##### ヘッダー行

| 要素 | 詳細 |

| ボタン | 詳細 |

| 「SEND NEUTRALIZED ▶」 | プライマリボタン。大文字、11px、ボールド、`#00D4FF`背景 + `#0D0D0D`テキスト。右に `send` アイコン |

- **配置**: 右寄せ（`justify-end`）
- **「SEND NEUTRALIZED」タップ時**: 変換後テキストを相手に送信

##### トレイ全体のスタイル

| 要素 | 詳細 |
|------|------|
| 背景色 | `#242424`（elevate背景）+ 半透明 + ブラー（backdrop-blur） |
| 角丸 | 24px（上部のみ、または全方向） |
| ボーダー | 1px、`#333333`（極細） |
| 影 | 上方向のドロップシャドウ |
| 位置 | 画面下部固定、インプットドックの上に重なる |
| 横幅 | 画面幅の95%、中央寄せ |
| パディング | 24px |

#### 4. インプットドック（固定フッター）

画面最下部に固定される入力エリア。

| 要素 | 位置 | 詳細 |
|------|------|------|
| 添付ファイルアイコン | 左端 | `attach_file` アイコン、`#A0A0A0` |
| テキスト入力フィールド | 中央（flex-grow） | カプセル型、プレースホルダー「Draft your message...」 |
| マイクアイコン | 入力フィールド内右端 | `mic` アイコン、`#555555`（将来的な音声入力用） |
| 「CONFIRM」ボタン | 右端 | 大文字、12px、ボールド、レタースペーシング広め + `check_circle`（FILL=1）アイコン |

- **入力フィールド背景**: `#1A1A1A`
- **入力フィールド枠線**: 1px、`#333333`、フォーカス時 `#00D4FF` の40%不透明度
- **CONFIRMボタン**: `#00D4FF`背景、`#0D0D0D`テキスト、高さ48px、カプセル型
- **ドック背景**: `#0D0D0D` + ブラー効果（backdrop-blur）
- **上部ボーダー**: 1px、`#242424`
- **下部パディング**: safe area対応（32px）

### 送信フロー（Emoffコア機能）

```
1. ユーザーがテキスト入力欄にメッセージを入力
2. 「CONFIRM」ボタンをタップ
3. AI変換プレビュートレイがスライドアップで表示
   - 左列: Original Draft（ユーザーの原文をイタリックで表示）
   - 右列: Emoff-ed Version（AI変換後のクリーンなテキスト）
4a. 「SEND NEUTRALIZED」をタップ → 変換後テキストのみ相手に送信
4b. 「REFINE AI」をタップ → AIに再変換を指示（プロンプト調整可能）
4c. 「✕」をタップ → プレビューを閉じ、入力に戻る
```

### インタラクション

| アクション | 動作 |
|------------|------|
| 戻るボタンをタップ | トーク一覧画面へ戻る |
| 検索アイコンをタップ | トーク内メッセージ検索モード |
| `more_vert` をタップ | オプションメニュー（グループ管理等） |
| メッセージ入力 → 「CONFIRM」 | AI変換プレビュートレイを表示 |
| 「SEND NEUTRALIZED」 | 変換後テキストを送信、トレイを閉じる |
| 「REFINE AI」 | AI再変換を実行、プレビュー更新 |
| トレイの「✕」 | プレビューを閉じる |
| 添付ファイルアイコン | ファイル添付メニュー表示 |

### デザインノート

- **「NEUTRALIZED」バッジ**: AI変換済みメッセージであることを視覚的に示す重要なUI要素。すべての送信メッセージに付与される
- **比較レイアウト**: Original vs Emoff-ed の2カラム比較は、ユーザーにAI変換の効果を明確に理解させる。左はイタリック（感情的）、右はミディアムウェイト（客観的）でフォントウェイトでも差を表現
- **トレイのブラー背景**: 背後のメッセージが透けて見える半透明効果で、チャットのコンテキストを維持
- **「CONFIRM」vs「送信」**: 直接送信ではなく「確認」を経由する設計が、Emoffの哲学（感情的な即座の反応を防ぐ）を体現
- **タイポグラフィ一貫性**: ラベル・バッジ・ボタンすべてが大文字+レタースペーシング広めで統一
