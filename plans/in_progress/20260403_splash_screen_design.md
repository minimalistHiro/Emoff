# スプラッシュ画面 — Emotion off.

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象画面**: スプラッシュ画面

---

## 画面概要

アプリ起動時に最初に表示される画面。Emoffのブランドロゴを表示しながら、Firebase Authの認証状態をバックグラウンドで判定し、認証済みならホーム画面、未認証ならログイン画面へ自動遷移する。ユーザー操作は不要なパッシブ画面。

## 画面構成（上から順）

### 1. ブランドロゴ領域（画面中央）

| 要素 | 詳細 |
|------|------|
| アプリ名「EMOFF」 | 大文字・ボールド・レタースペーシング広め（tracking-widest）、32px |
| 配置 | 画面の垂直・水平中央 |

- **テキスト色**: `#00D4FF`（シアン）
- **下マージン**: 12px

### 2. タグライン

| 要素 | 詳細 |
|------|------|
| テキスト | 「Emotion off.」— 通常ウェイト、14px |
| 配置 | ロゴの直下、中央寄せ |

- **テキスト色**: `#A0A0A0`（副テキスト）
- **下マージン**: 48px

### 3. ローディングインジケーター

| 要素 | 詳細 |
|------|------|
| タイプ | 回転スピナー |
| サイズ | 24px |
| 配置 | タグラインの下、中央寄せ |

- **実装時**: `CustomLoadingIndicator` を使用
- **色**: `#00D4FF`（シアン）

## 背景演出

| 要素 | 詳細 |
|------|------|
| メイン背景 | `#0D0D0D`（ダーク） |
| アンビエントグロー（任意） | 画面中央にシアンの大きなぼかし円（blur: 150px、opacity: 3〜5%）を配置し、ロゴが浮かび上がるような演出 |

## フェードイン演出

| 要素 | 詳細 |
|------|------|
| ロゴ+タグライン | 画面表示時に `FadeIn` + 軽い `ScaleUp`（0.95→1.0）アニメーション、duration: 600ms |
| ローディングインジケーター | ロゴアニメーション完了後に `FadeIn`、duration: 300ms |

## インタラクション

| アクション | 動作 |
|------------|------|
| 画面表示時 | Firebase Auth の `currentUser` を確認（認証状態の判定） |
| 認証済み | ホーム画面へ遷移（`pushReplacement`） |
| 未認証 | ログイン画面へ遷移（`pushReplacement`） |
| 最低表示時間 | 1.5秒（認証判定が高速でも、ブランド訴求のため最低限表示する） |
| 認証判定エラー | ログイン画面へ遷移（安全側に倒す） |

## 遷移仕様

| 条件 | 遷移先 | 遷移方法 |
|------|--------|---------|
| `FirebaseAuth.instance.currentUser != null` | ホーム画面 | `pushReplacement` + フェード遷移 |
| `currentUser == null` | ログイン画面 | `pushReplacement` + フェード遷移 |
| 例外発生 | ログイン画面 | `pushReplacement` + フェード遷移 |

## デザインノート

- **ミニマル構成**: ロゴ・タグライン・インジケーターの3要素のみ。Emoffの「感情を排す」哲学をアプリ起動時点から体現する
- **「Emotion off.」**: タグラインはアプリ名の由来とコンセプトを端的に伝えるフレーズ。ピリオドで終わることで宣言的なトーンを演出
- **最低表示時間1.5秒**: 認証判定はほぼ瞬時だが、ロゴの印象づけとフェードイン演出を見せるために最低限の表示時間を確保する
- **pushReplacement**: スプラッシュ画面は起動時限定のため、ナビゲーションスタックに残さない
