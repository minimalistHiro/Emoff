# グループ作成画面 — Assemble Your Circle

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象画面**: グループ作成画面

---

## 画面概要

友達リストからメンバーを選択し、新しいグループチャットを作成する画面。トーク一覧画面の「New Discussion」ボタンから遷移する。
グループチャットは **Personal Pro以上** の機能のため、Freeプランユーザーがアクセスした場合はアップグレード促進ダイアログを表示する。

**対応Firestoreデータ:**
- `chats` コレクション（`type: "group"`, `members`, `groupName`, `groupIconUrl`, `createdBy`）
- `users/{uid}/friends` サブコレクション（メンバー選択候補）

---

## Freeプランユーザーのアクセス制御

Freeプランユーザーが「New Discussion」ボタンをタップした場合、グループ作成画面へは遷移せず、**アップグレード促進ダイアログ**を表示する。

| 要素 | 詳細 |
|------|------|
| アイコン | `groups` Material Icon、48px、`#00D4FF` |
| タイトル | 「グループチャットはProプランの機能です」— ボールド、18px、`#FFFFFF` |
| 説明テキスト | 「Personal Proプラン（¥300/月）にアップグレードすると、グループチャット・変換トーン選択などの機能が利用できます。」— 14px、`#A0A0A0` |
| アップグレードボタン | 「プランを見る」— `CustomButton`（`primary`バリアント） |
| 閉じるボタン | 「あとで」— `CustomButton`（`secondary`バリアント） |

- **実装**: `CustomDialog` を使用（UI_UX.mdルール準拠）
- **「プランを見る」タップ時**: サブスクリプション/課金画面へ遷移（未設計。暫定的にダイアログを閉じる）

## 画面構成（上から順）— Pro以上のユーザー

### 1. AppBar（固定ヘッダー）

| 要素 | 位置 | 詳細 |
|------|------|------|
| 戻るボタン | 左端 | `arrow_back_ios` アイコン、`#00D4FF`。トーク一覧画面に戻る |
| 画面タイトル | 中央 | 「New Group」— セミボールド、18px、`#FFFFFF` |
| 作成ボタン | 右端 | 「Create」テキストボタン、`#00D4FF`、セミボールド、16px。有効/無効状態あり |

- **実装時**: `CustomAppBar` を使用
- **背景色**: `#0D0D0D`
- **作成ボタン無効時**: テキスト色 `#555555`（タップ不可）
- **作成ボタン有効条件**: グループ名が1文字以上 AND メンバーが1人以上選択済み

### 2. グループ情報セクション

| 要素 | 詳細 |
|------|------|
| グループアイコン | 80×80px、円形。デフォルトは `groups` Material Icon（`#A0A0A0`）+ `#242424`背景。中央にカメラオーバーレイ（`camera_alt`、24px、`#00D4FF`、半透明黒背景の小円） |
| グループ名入力 | アイコンの右側に配置。プレースホルダー「グループ名を入力」 |

- **アイコンタップ時**: 画像選択（カメラ/ギャラリー）のボトムシートを表示。アイコン設定は任意（未設定でもグループ作成可能）
- **グループ名入力**: `CustomTextField` を使用。最大文字数: 50文字。カウンター表示あり
- **レイアウト**: 横並び（Row）。アイコン左、入力フィールド右（Expanded）
- **背景色**: `#1A1A1A`（カード背景）
- **パディング**: 上下20px、左右16px
- **下部**: 1pxの区切り線（`#242424`）

### 3. 選択済みメンバープレビュー

メンバーが1人以上選択されている場合に表示。未選択時は非表示。

| 要素 | 詳細 |

- **上下パディング**: 16px
- **左右パディング**: 16px

#### 友達アイテム1件あたりの構成

| 要素 | 詳細 |

## インタラクション

| アクション | 動作 |
|------------|------|
| 戻るボタンをタップ | トーク一覧画面に戻る。入力中の場合は破棄確認ダイアログ（`CustomDialog`）を表示 |
| グループアイコンをタップ | 画像選択ボトムシート（カメラ撮影 / ギャラリーから選択 / キャンセル）。背景色 `#242424` |
| グループ名を入力 | リアルタイムで文字数カウンター更新。50文字上限 |
| 友達アイテムをタップ | 選択/解除をトグル。選択時: アバターがカラー化 + チェックマークがシアンに。セクション3のメンバープレビューに即座に反映 |
| メンバーチップの×をタップ | 該当メンバーの選択を解除。友達リストのチェックも連動して解除 |
| 検索バーに入力 | 友達リストをリアルタイムフィルタリング |
| 「Create」/ 「グループを作成」をタップ | グループ作成処理を実行（後述） |

### グループ作成処理フロー

1. ボタンタップ → ボタンをローディング状態に変更（`CustomLoadingIndicator`）
2. Firestoreに `chats` ドキュメントを作成:
   - `type`: `"group"`
   - `members`: [自分のUID, 選択メンバーのUID...]
   - `groupName`: 入力されたグループ名
   - `groupIconUrl`: 設定された画像URL（未設定の場合は `null`）
   - `createdBy`: 自分のUID
   - `createdAt`: サーバータイムスタンプ
   - `updatedAt`: サーバータイムスタンプ
3. 成功時 → 作成したグループのトークルーム画面へ遷移（`pushReplacement`でスタックを置き換え）
4. 失敗時 → エラーダイアログ表示（`CustomDialog` + `toUserFriendlyError()`）

### 破棄確認ダイアログ

グループ名が入力済み、またはメンバーが1人以上選択済みの状態で戻るボタンをタップした場合に表示。

| 要素 | 詳細 |
|------|------|
| タイトル | 「作成を中止しますか？」— ボールド、18px、`#FFFFFF` |
| 説明 | 「入力した内容は保存されません。」— 14px、`#A0A0A0` |
| 「中止する」ボタン | `CustomButton`（`danger`バリアント） |
| 「続ける」ボタン | `CustomButton`（`secondary`バリアント） |

## デザインノート

- **コンセプト「Assemble Your Circle」**: 「Circle of Clarity」（ホーム画面）の延長として、信頼できるメンバーでグループを組成するイメージ
- **2段階の作成ボタン**: AppBar右端のテキストボタンは片手操作時のアクセス用、画面下部の固定ボタンはスクロール後の操作用。どちらも同一の有効条件・処理
- **グレースケール→カラー**: ホーム画面と同様、未選択のアバターはグレースケール。選択するとカラーに変化し、「このメンバーをCircleに加えた」ことを視覚的にフィードバック
- **グループアイコンは任意**: 初回作成時はアイコン未設定でOK。デフォルトのMaterial Iconが適用される。後からグループ管理画面で設定・変更可能
- **作成後の遷移先**: グループ管理画面ではなく、トークルーム画面へ直接遷移する。すぐに会話を始められる体験を優先。グループ管理はトークルームのメニューからアクセス可能
- **メンバー上限**: Firestoreの `members` 配列は上限目安500人だが、UIとしては特に上限表示をしない。友達リストの人数が自然な制約となる
