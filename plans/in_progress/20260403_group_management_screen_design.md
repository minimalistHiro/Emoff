# グループ管理画面 — Circle Settings

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象画面**: グループ管理画面

---

## 画面概要

グループチャットの情報確認・編集・メンバー管理を行う画面。トークルーム画面の `more_vert` メニューから「グループ管理」を選択して遷移する。
グループ作成者（`createdBy`）と一般メンバーで操作権限が異なる。

**対応Firestoreデータ:**
- `chats` コレクション（`type: "group"`, `members`, `groupName`, `groupIconUrl`, `createdBy`）
- `users/{uid}/friends` サブコレクション（メンバー追加時の候補）
- `users` コレクション（メンバーのプロフィール情報取得）

---

## 権限モデル

| 操作 | 作成者（`createdBy`） | 一般メンバー |
|------|:---:|:---:|
| グループ名の編集 | ○ | × |
| グループアイコンの変更 | ○ | × |
| メンバーの追加 | ○ | ○ |
| メンバーの削除（退会させる） | ○ | × |
| グループから退出 | ○ | ○ |

- 作成者が退出した場合、`createdBy` は履歴データとして残る。残りのメンバーには編集・削除権限がなくなる（メンバー追加・退出のみ可能）
- 全メンバーが退出した場合、`chats` ドキュメントは残存するが機能しない（将来的にCloud Functionsで定期クリーンアップ検討）

3. 残りは名前の昇順（A→Z / あ→ん）

## ボトムシート: メンバー追加

「メンバーを追加」タップ時に表示されるボトムシート。グループ作成画面のメンバー選択UIを簡略化した構成。

### ボトムシート構成

| 要素 | 詳細 |
|------|------|
| ハンドルバー | 中央、40×4px、角丸フル、`#555555` |
| タイトル | 「メンバーを追加」— ボールド、18px、`#FFFFFF` |
| 検索バー | `CustomTextField`、プレースホルダー「友達を検索...」、カプセル型 |
| 友達リスト | チェック式。既存メンバーはグレーアウト+「参加中」ラベル表示でタップ不可 |
| 追加ボタン | 「追加する」— `CustomButton`（`primary`バリアント）。固定フッター |

- **ボトムシート背景色**: `#242424`
- **角丸**: 上部24px
- **高さ**: 画面の70%
- **友達リストのアイテム構成**: グループ作成画面のセクション5と同一仕様

#### 友達リスト — 既存メンバーの表示

| 要素 | 詳細 |
|------|------|
| アバター画像 | 48×48px、円形。グレースケール（50%）— 通常より濃いグレー |
| 名前 | 16px、`#555555`（無効色） |
| ステータス | 名前の下、13px、`#555555`。「参加中」 |
| チェックマーク | 非表示（タップ不可） |

#### 追加処理フロー

1. メンバーを選択して「追加する」をタップ
2. ボタンをローディング状態に変更
3. Firestoreの `chats` ドキュメントの `members` 配列に新メンバーのUIDを追加
4. 成功時 → ボトムシートを閉じ、メンバーリストに即座に反映
5. 失敗時 → エラーダイアログ表示（`CustomDialog` + `toUserFriendlyError()`）

## ダイアログ

### メンバー削除確認ダイアログ

作成者がメンバーの削除ボタンをタップした場合に表示。

| 要素 | 詳細 |
|------|------|
| タイトル | 「メンバーを削除しますか？」— ボールド、18px、`#FFFFFF` |
| 説明 | 「{名前}をこのグループから削除します。」— 14px、`#A0A0A0` |
| 「削除する」ボタン | `CustomButton`（`danger`バリアント） |
| 「キャンセル」ボタン | `CustomButton`（`secondary`バリアント） |

- **実装**: `CustomDialog` を使用
- **削除処理**: Firestoreの `members` 配列から対象UIDを除去

### グループ退出確認ダイアログ

| 要素 | 詳細 |
|------|------|
| タイトル | 「グループを退出しますか？」— ボールド、18px、`#FFFFFF` |
| 説明（一般メンバー） | 「このグループから退出します。再度参加するにはメンバーからの招待が必要です。」— 14px、`#A0A0A0` |
| 説明（作成者） | 「あなたはこのグループの作成者です。退出すると、グループ名やアイコンの編集・メンバーの削除ができなくなります。」— 14px、`#A0A0A0` |
| 「退出する」ボタン | `CustomButton`（`danger`バリアント） |
| 「キャンセル」ボタン | `CustomButton`（`secondary`バリアント） |

- **実装**: `CustomDialog` を使用
- **退出処理フロー**:
  1. Firestoreの `members` 配列から自分のUIDを除去
  2. 成功時 → トーク一覧画面へ遷移（`pushAndRemoveUntil` でスタックをクリア）
  3. 失敗時 → エラーダイアログ表示

## デザインノート

- **コンセプト「Circle Settings」**: 「Assemble Your Circle」（グループ作成画面）で組成したグループの管理・調整を行う場
- **権限の視覚的明示**: OWNERバッジ・YOUバッジで役割を一目で把握可能。編集アイコンやカメラオーバーレイは作成者にのみ表示し、一般メンバーには不要なUIを見せない
- **インライン編集**: グループ名の編集は別画面に遷移せず、その場で即座に変更可能。ミニマルな操作ステップ
- **メンバー追加のボトムシート**: フルスクリーン遷移ではなくボトムシートを採用。グループ管理画面のコンテキストを維持しつつ、すばやくメンバーを追加できる
- **退出時の作成者警告**: 作成者が退出すると編集・削除権限が失われることを明示的に伝える。ただし退出自体はブロックしない（「あなたしか操作できなくなります」ではなく「操作できなくなります」）
- **グレースケールの一貫性**: メンバーアバターもホーム画面・グループ作成画面と同様のグレースケール（20%）を適用し、アプリ全体のトーンを統一
