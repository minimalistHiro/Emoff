# トークルーム: 通報フローUI

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象画面**: トークルーム画面（既存設計への追加）
- **親設計書**: `plans/in_progress/20260403_talk_room_screen_design.md`

---

## 概要

トークルーム画面内でユーザーまたは特定メッセージを通報するためのUI。2つの導線からアクセスでき、通報理由の選択→補足入力→送信の3ステップで完結する。

**対応Firestoreデータ:**
- `reports` コレクション（`reporterUid`, `targetUid`, `reason`, `detail`, `messageId`, `chatId`）

---

## 導線

### 導線1: メニューからの通報（ユーザー通報）

トークルーム AppBar の `more_vert` メニューに「通報する」項目を追加。

| メニュー項目 | アイコン | テキスト色 | 備考 |
|-------------|---------|-----------|------|
| グループ管理 | `group` | `#FFFFFF` | グループチャットのみ表示（既存） |
| 通報する | `flag` | `#FF4D4D` | メニュー最下部に配置 |

- **メニュースタイル**: 背景 `#242424`、角丸12px、アイテム高さ48px、パディング左右16px
- **タップ時**: 通報理由選択ボトムシートを表示。`targetUid` = チャット相手のUID。`messageId` = null（特定メッセージなし）
- **グループチャットの場合**: メニューに「通報する」は表示しない（メッセージ長押しからの個別通報のみ）

### 導線2: メッセージ長押しからの通報（メッセージ通報）

受信メッセージ（相手のバブル）を長押しした際にコンテキストメニューを表示。

| メニュー項目 | アイコン | テキスト色 |
|-------------|---------|-----------|
| このメッセージを通報 | `flag` | `#FF4D4D` |

- **長押し対象**: 受信メッセージのみ（自分の送信メッセージは長押しメニュー非表示）
- **コンテキストメニュースタイル**: メッセージバブル上部にフローティング表示。背景 `#242424`、角丸12px、影あり
- **タップ時**: 通報理由選択ボトムシートを表示。`targetUid` = メッセージ送信者のUID。`messageId` = 対象メッセージのID
- **長押しフィードバック**: バブル背景が `#333333` に変化（200ms）+ 軽い触覚フィードバック（HapticFeedback.mediumImpact）

## 通報理由選択ボトムシート

### 構成

| 要素 | 詳細 |
|------|------|
| ハンドルバー | 中央、40×4px、角丸フル、`#555555` |
| タイトル | 「通報する」— ボールド、18px、`#FFFFFF` |
| 説明テキスト | 「該当する理由を選択してください。通報内容は相手に通知されません。」— 14px、`#A0A0A0` |
| 理由リスト | ラジオボタン式（後述） |
| 補足入力欄 | 条件付き表示（後述） |
| 送信ボタン | 「通報を送信」— `CustomButton`（`danger`バリアント）。固定フッター |

- **ボトムシート背景色**: `#242424`
- **角丸**: 上部24px
- **パディング**: 上16px、左右16px、下（SafeArea分 + 16px）

### 理由リスト

FIRESTORE.md の `reason` フィールドに対応する4つの選択肢。

| 値 | 表示テキスト | アイコン |
|----|------------|---------|
| `spam` | スパム・迷惑行為 | `block` |
| `harassment` | 嫌がらせ・いじめ | `warning` |
| `inappropriate_content` | 不適切なコンテンツ | `visibility_off` |
| `other` | その他 | `more_horiz` |

**アイテムスタイル:**
- 各アイテム: 左にアイコン（24px、`#A0A0A0`）、中央にテキスト（16px、`#FFFFFF`）、右にラジオボタン
- ラジオボタン（未選択）: 円形枠線（24×24px、2px、`#555555`）
- ラジオボタン（選択済み）: シアン枠線（`#00D4FF`）+ 内側に塗りつぶし円（12×12px、`#00D4FF`）
- アイテム高さ: 56px
- タップ領域: アイテム全体

### 補足入力欄

`other`（その他）を選択した場合のみ表示。それ以外の理由では非表示。

| 要素 | 詳細 |
|------|------|
| ラベル | 「詳細を入力してください」— 13px、`#A0A0A0` |
| テキスト入力 | `CustomTextField`、複数行（maxLines: 4）、プレースホルダー「具体的な内容を記入...」、最大500文字 |

- **表示アニメーション**: フェードイン + スライドダウン（300ms）

### 送信ボタンの有効条件

- 理由が1つ選択されていること
- `other` 選択時は、補足テキストが1文字以上入力されていること

## 通報送信処理フロー

```
1. 「通報を送信」タップ → ボタンをローディング状態に変更
2. Firestore `reports` コレクションにドキュメントを作成:
   - reporterUid: 自分のUID
   - targetUid: 通報対象のUID
   - reason: 選択された理由
   - detail: 補足テキスト（other以外はnull）
   - messageId: メッセージ長押し経由の場合のみ（メニュー経由はnull）
   - chatId: 現在のチャットID
   - createdAt: サーバータイムスタンプ
3. 成功時 → 完了ダイアログを表示
4. 失敗時 → エラーダイアログ表示（CustomDialog + toUserFriendlyError()）
```

### 完了ダイアログ

| 要素 | 詳細 |
|------|------|
| アイコン | `check_circle`、48px、`#00E676`（成功色） |
| タイトル | 「通報を受け付けました」— ボールド、18px、`#FFFFFF` |
| 説明 | 「内容を確認のうえ、必要に応じて対応いたします。」— 14px、`#A0A0A0` |
| 閉じるボタン | 「閉じる」— `CustomButton`（`secondary`バリアント） |

- **実装**: `CustomDialog` を使用
- **閉じるタップ時**: ダイアログとボトムシートの両方を閉じ、トークルームに戻る

## デザインノート

- **2つの導線**: メニューからの「ユーザー通報」とメッセージ長押しからの「メッセージ通報」を用意。メッセージ通報では `messageId` が紐づくため、管理者が具体的な対象を特定できる
- **通報は匿名**: 「通報内容は相手に通知されません」を明記し、通報のハードルを下げる
- **dangerバリアント**: 送信ボタンを赤にすることで、通常のアクション（送信・保存）とは異なる重要性を視覚的に伝える
- **グループでの通報**: グループチャットではメニューからの一括通報は複雑なため、メッセージ長押しによる個別通報のみとする
- **補足入力の条件付き表示**: 「その他」選択時のみ入力欄を表示することで、通常の通報フローをシンプルに保つ
