# 通知設定画面 — Notifications

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象画面**: 通知設定画面

---

## 画面概要

プッシュ通知のON/OFFおよび通知種別ごとの設定を行う画面。
設定画面の「Notifications」から遷移する。
通知設定はデバイスローカル（SharedPreferences）に保存し、FCMトークンの登録/解除と連動する。

## データ保存

### 保存先: SharedPreferences（デバイスローカル）

| キー | 型 | デフォルト値 | 説明 |
|------|-----|------------|------|
| `notification_enabled` | bool | `true` | 全通知の一括ON/OFF |
| `notification_messages` | bool | `true` | 新着メッセージ通知 |
| `notification_friend_requests` | bool | `true` | 友達申請通知 |
| `notification_announcements` | bool | `true` | お知らせ通知 |

- 初回起動時は全て `true`（ON）
- Firestoreには保存しない（端末ごとの設定）

### FCMとの連動

| 操作 | FCM処理 |
|------|---------|
| メイントグルをOFF | FCMトークンをサーバーから削除（通知が一切届かなくなる） |
| メイントグルをON | FCMトークンを再登録 |
| 個別トグルの変更 | FCMトークンはそのまま。サーバー側（Cloud Functions）で通知送信時に設定値を参照してフィルタリング |

※ FCMの実装詳細はR-10の決定後に確定。本設計ではUI層の仕様のみ定義する。

## インタラクション

| アクション | 動作 |
|------------|------|
| 戻るボタンをタップ | 設定画面に戻る |
| メイントグルをOFF | 個別設定を全て非活性化（opacity: 0.4）。FCMトークン削除 |
| メイントグルをON | 個別設定を活性化。FCMトークン再登録。端末の通知権限がない場合は権限リクエストダイアログを表示 |
| 個別トグルを変更 | 即座にSharedPreferencesに保存。UIにフィードバックなし（トグルの切替アニメーションのみ） |

### 通知権限リクエスト

メイントグルをONにした際、端末の通知権限が未許可の場合:

```
[メイントグルをON]
  ├── 端末の通知権限チェック
  ├── 未許可 → 権限リクエストダイアログ（OS標準）
  │     ├── 許可 → トグルON、FCMトークン登録
  │     └── 拒否 → ダイアログ表示:「端末の設定から通知を許可してください」
  │                 トグルをOFFに戻す
  └── 許可済み → トグルON、FCMトークン登録
```

## エッジケース

| ケース | 対応 |
|--------|------|
| 端末の通知権限がOFF | メイントグルをONにしようとした際にOS権限リクエスト。拒否されたらトグルをOFFに戻す |
| 端末設定で後から通知を無効化 | アプリ側のトグルはONのまま。注意テキスト（セクション4）で端末設定の確認を案内 |
| FCMトークン登録失敗 | エラーダイアログ:「通知の設定に失敗しました」。トグルを元の状態に戻す |
| メイントグルOFF中に個別トグルをタップ | 非活性化されているためタップ不可 |
| 初回起動時 | 全トグルON。ただし端末の通知権限リクエストはアプリ初回起動時に別途行う |
| ネットワークエラー（FCMトークン操作時） | エラーダイアログ:「通信エラーが発生しました」。トグルを元の状態に戻す |

## デザインノート

- **メイントグルの支配関係**: メイントグルOFFで個別設定を非活性化する「マスタースイッチ」パターン。一括OFFしたいユーザーと細かく制御したいユーザーの両方に対応
- **設定の即時保存**: 保存ボタンなしで、トグル操作と同時にSharedPreferencesに反映。設定画面のUXとして自然な振る舞い
- **Firestoreに保存しない理由**: 通知設定はデバイス固有（同じアカウントでもデバイスごとに異なる設定が自然）。端末ローカル保存で十分
- **シンプルな3項目**: 通知種別を Messages / Friend Requests / Announcements の3つに絞り、設定の複雑さを最小限に。将来的に通知種別が増えた場合もこのセクションに追加するだけで拡張可能
- **注意テキスト**: 端末のOS設定で通知を無効にしているケースへの案内。目立たない色（`#555555`）で補足的に配置
- **R-10との関係**: FCMの実装詳細（通知ペイロードの内容、バッジ管理等）はR-10で決定する。本設計はUI層の仕様に限定し、実装フェーズでFCM層と結合する
