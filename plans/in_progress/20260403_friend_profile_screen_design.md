# 友達プロフィール画面 — Peer Profile

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象画面**: 友達プロフィール画面 + ブロック確認ダイアログ

---

## 画面概要

友達の背景画像・アイコン・アカウント情報を表示するプロフィール画面。
LINEのプロフィール画面に相当し、下部にトークボタンを配置する。
ホーム画面の友達リストからカードをタップして遷移する。

---

## 画面構成（上から順）

### 1. AppBar（透過ヘッダー）

| 要素 | 位置 | 詳細 |
|------|------|------|
| 戻るアイコン | 左端 | `arrow_back_ios` アイコン。ホーム画面に戻る |
| メニューアイコン | 右端 | `more_vert` アイコン。ポップアップメニューを表示 |

- **実装時**: `CustomAppBar` を使用。`backgroundColor: Colors.transparent`、`elevation: 0`
- **アイコン色**: `#FFFFFF`（白。背景画像上で視認性確保のため）
- **アイコンに影**: 軽いドロップシャドウ（`Shadow(blurRadius: 4, color: Colors.black54)`）で背景画像上の視認性を確保
- **背景**: 透過（背景画像の上にオーバーレイ）

**ポップアップメニュー項目:**

| 項目 | アイコン | テキスト色 | 動作 |
|------|---------|-----------|------|
| ブロック | `block` | `#FF4D4D`（エラー色） | ブロック確認ダイアログを表示 |
| 通報 | `flag` | `#A0A0A0` | 通報フロー（フェーズ3で設計。暫定で項目のみ表示） |

- **メニュー背景**: `#242424`
- **メニュー角丸**: 12px

### 2. 背景画像エリア

| 要素 | 詳細 |
|------|------|
| 背景画像 | 画面上部全幅、高さ 240px。`users.backgroundUrl` を表示 |
| デフォルト | 未設定時はグラデーション背景（`#1A1A1A` → `#242424`、上から下） |
| オーバーレイ | 黒の半透明グラデーション（下部が濃い）。アイコンとの境界を自然にする |

- **実装時**: `Stack` で背景画像 + グラデーションオーバーレイを重ねる
- **画像フィット**: `BoxFit.cover`

### 3. プロフィールアイコン

| 要素 | 詳細 |
|------|------|
| アイコン画像 | 96×96px、円形、白ボーダー（2px、`#FFFFFF`） |
| 配置 | 中央寄せ、背景画像の下端から半分はみ出す形（`top: 192px` 相当） |
| デフォルト | 未設定時は `person` アイコン（48px、`#555555`）+ `#1A1A1A` 背景 |

- **実装時**: `Stack` + `Positioned` で背景画像に重ねて配置
- **シャドウ**: 軽いドロップシャドウ（`elevation: 4` 相当）

### 4. ユーザー情報セクション

| 要素 | 詳細 |
|------|------|
| ユーザー名 | ボールド、24px、`#FFFFFF`、中央寄せ |
| ユーザーID | `@userId` 形式、14px、`#A0A0A0`、中央寄せ |
| 区切り線 | 1px、`#242424`、上下マージン 16px、左右マージン 48px |
| 自己紹介文 | 通常ウェイト、14px、`#A0A0A0`、中央寄せ、最大5行 |

- **配置**: アイコンの下、中央寄せ
- **アイコンとの間隔**: 16px
- **自己紹介文**: 未設定時は非表示（区切り線も非表示）
- **左右パディング**: 32px

### 5. トークボタン（メインアクション）

| 要素 | 詳細 |
|------|------|
| ボタンテキスト | 「トーク」 |
| アイコン | `chat_bubble_outline`、ボタンテキストの左に配置 |
| バリアント | `primary`（シアン背景 + 黒テキスト） |

- **実装時**: `CustomButton`（primary バリアント）
- **配置**: 画面下部に固定（`bottomNavigationBar` の上）
- **左右マージン**: 24px
- **下マージン**: ボトムナビの上に 16px の間隔
- **高さ**: 48px（CustomButton標準）

### 6. ボトムナビゲーション（共通コンポーネント）

ホーム画面と同様の構成。FRIENDSタブがアクティブ状態。

## ブロック確認ダイアログ（2-3a）

AppBarのメニュー → 「ブロック」をタップした際に表示。

### ダイアログ構成

| 要素 | 詳細 |
|------|------|
| アイコン | `block`、48px、`#FF4D4D` |
| タイトル | 「〇〇さんをブロック」— ボールド、18px、`#FFFFFF` |
| 説明文 | 以下の3点を箇条書きで表示、14px、`#A0A0A0` |
| キャンセルボタン | 「キャンセル」— `CustomButton`（secondary バリアント） |
| ブロックボタン | 「ブロックする」— `CustomButton`（danger バリアント） |

**説明文の内容:**
- 「相手からのメッセージが届かなくなります」
- 「友達リストからお互いに削除されます」
- 「ブロックしたことは相手に通知されません」

- **実装時**: `CustomDialog` を使用
- **背景色**: `#242424`
- **角丸**: 16px
- **ボタン配置**: 横並び（左: キャンセル、右: ブロック）、ボタン間隔 12px

### ブロック処理フロー

```
[ブロックボタンタップ]
  ├── ローディング表示（ダイアログ内ボタンを非活性化 + CustomLoadingIndicator）
  ├── 処理実行:
  │     ① users/{myUid}/blocked_users/{targetUid} にドキュメント作成
  │     ② Cloud Functionsが以下を自動処理:
  │          - 両者の friends サブコレクションから相互削除
  │          - （チャットは削除しない。メッセージ非表示はクライアント側で制御）
  ├── 成功 → ダイアログ閉じる → ホーム画面に戻る（友達リストから消えるため）
  └── 失敗 → エラーダイアログ表示（toUserFriendlyError で変換）
```

## Firestoreとの対応

| UI要素 | Firestoreクエリ/操作 |
|--------|---------------------|
| 背景画像 | `users/{friendUid}.backgroundUrl` |
| アイコン画像 | `users/{friendUid}.iconUrl` |
| ユーザー名 | `users/{friendUid}.name` |
| ユーザーID | `users/{friendUid}.userId` |
| 自己紹介文 | `users/{friendUid}.bio` |
| トークボタン | `users/{myUid}/friends/{friendUid}.chatId` が存在すれば既存チャットを開く。なければ新規 `chats` ドキュメントを作成して遷移 |
| ブロック処理 | `users/{myUid}/blocked_users/{friendUid}` に `{ createdAt: now }` を作成 |

## インタラクション

| アクション | 動作 |
|------------|------|
| 戻るアイコンをタップ | ホーム画面に戻る（`Navigator.pop`） |
| メニューアイコンをタップ | ポップアップメニュー表示（ブロック / 通報） |
| 「ブロック」をタップ | ブロック確認ダイアログ表示 |
| 「通報」をタップ | 通報フロー（フェーズ3で実装。暫定でタップ不可またはダイアログ「準備中です」） |
| トークボタンをタップ | トークルーム画面へ遷移。既存チャットがあればそれを開き、なければ新規作成 |
| 背景画像をタップ | 何もしない |
| アイコンをタップ | 何もしない |

## エッジケース

| ケース | 対応 |
|--------|------|
| 背景画像未設定 | グラデーション背景（`#1A1A1A` → `#242424`）を表示 |
| アイコン未設定 | デフォルトアバター（`person` アイコン）を表示 |
| 自己紹介文未設定 | 区切り線と自己紹介文セクションを非表示 |
| 相手がアカウント削除済み | データ取得時にドキュメント不存在 → エラーダイアログ「このユーザーは存在しません」→ ホーム画面に戻る |
| ネットワークエラー | エラーダイアログ:「通信エラーが発生しました」 |
| トーク開始時のチャット重複 | `friends/{friendUid}.chatId` を正として使用。未作成時のみ新規作成し、即座に `chatId` を `friends` ドキュメントに書き戻す |
| 自己紹介文が長い | 最大5行で `overflow: ellipsis` |

## デザインノート

- **LINEライクなレイアウト**: 背景画像 + 重なるアイコン + 下部アクションボタンの構成はLINEのプロフィール画面を踏襲。Emoffユーザーに馴染みやすい
- **アクションは「トーク」のみ**: 音声通話・ビデオ通話等のボタンは配置しない。Emoffの「テキストで伝える」コンセプトに集中
- **透過AppBar**: 背景画像を最大限活かすため、AppBarを透過にしてコンテンツの上にオーバーレイ。アイコンにドロップシャドウを付けて視認性確保
- **ブロック導線の慎重さ**: メニュー → ダイアログ → 確認ボタンの3ステップで誤操作を防止。ダイアログ内で影響を箇条書きで明示し、ユーザーが結果を理解した上で操作できるようにする
- **ブロック後の遷移**: ブロック成功後はホーム画面に自動的に戻る。友達リストからも消えているため、プロフィール画面に留まる意味がない
- **通報機能の暫定対応**: フェーズ3のトークルーム通報フローUI（3-a）の設計と合わせて実装するため、本画面では項目のみ表示
