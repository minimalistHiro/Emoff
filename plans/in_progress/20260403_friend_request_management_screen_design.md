# 友達申請管理画面 — Pending Connections

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象画面**: 友達申請管理画面

---

## 画面概要

受信した友達申請の一覧を表示し、承認・拒否を行う画面。
ホーム画面の友達申請管理ボタン（バッジ付き）から遷移する。
リアルタイムリスナーで申請の追加・変更を即時反映する。

## Firestoreとの対応

| UI要素 | Firestoreクエリ/操作 |
|--------|---------------------|
| 申請リスト取得 | `friend_requests` を `where('toUid', '==', myUid).where('status', '==', 'pending').orderBy('createdAt', 'desc')` でリアルタイムリスナー |
| アバター画像 | `friend_requests.fromIconUrl` |
| 申請者名 | `friend_requests.fromName` |
| 申請者ID | 別途 `users` コレクションから `fromUid` で取得し `userId` を表示 |
| 申請日時 | `friend_requests.createdAt` を相対時間に変換 |
| 件数バッジ | リスナーから取得したドキュメント数 |
| 承認ボタン | ① `friend_requests/{requestId}` の `status` を `accepted`、`updatedAt` を更新 ② 両者の `users/{uid}/friends/{friendUid}` にドキュメント作成（`name`, `iconUrl`, `createdAt`） |
| 拒否ボタン | `friend_requests/{requestId}` の `status` を `rejected`、`updatedAt` を更新 |

**注意**: 承認時の友達ドキュメント作成は双方向（自分と相手の両方の `friends` サブコレクション）。Cloud Functionsで処理するか、クライアント側バッチ書き込みで処理するかは実装時に決定。

## インタラクション

| アクション | 動作 |
|------------|------|
| 戻るアイコンをタップ | ホーム画面に戻る（`Navigator.pop`） |
| 承認ボタンをタップ | 確認ダイアログ:「〇〇さんの友達申請を承認しますか？」→ 確認で承認処理実行 |
| 承認成功 | カードがリストからフェードアウトで消失 + 成功ダイアログ:「〇〇さんと友達になりました」 |
| 拒否ボタンをタップ | 確認ダイアログ:「〇〇さんの友達申請を拒否しますか？」→ 確認で拒否処理実行 |
| 拒否成功 | カードがリストからフェードアウトで消失（ダイアログなし、静かに処理） |
| 承認/拒否失敗 | エラーダイアログ表示（`toUserFriendlyError` で変換） |
| 申請カードのアバター/名前をタップ | 何もしない（プロフィール画面への遷移は友達になった後にホーム画面から） |
| リストをプルダウン | リフレッシュ（リアルタイムリスナーが正常動作していれば手動リフレッシュは不要だが、フォールバックとして用意） |

## 状態遷移

```
[画面表示]
  ├── リスナー接続中 → CustomLoadingIndicator を中央に表示
  ├── 申請0件 → Empty State（セクション4）
  └── 申請あり → 申請リスト表示（セクション3）
        ├── 承認タップ → 確認ダイアログ → 承認処理 → カード消失 + 成功ダイアログ
        └── 拒否タップ → 確認ダイアログ → 拒否処理 → カード消失
```

## エッジケース

| ケース | 対応 |
|--------|------|
| ネットワークエラー | エラーダイアログ:「通信エラーが発生しました。ネットワーク接続を確認してください」 |
| 承認処理中に相手がアカウント削除 | Cloud Functions側でエラーハンドリング。クライアントにはエラーダイアログ表示 |
| 同じ相手から再申請（拒否後） | 新しい `friend_requests` ドキュメントとして表示される（拒否は永続ブロックではない） |
| 大量の申請（100件以上） | `limit(50)` でページネーション。スクロール末尾で追加読み込み |
| 承認/拒否ボタンの二重タップ | ボタンタップ後に即座に非活性化（ローディング状態にする）で防止 |

## デザインノート

- **リアルタイム更新**: `snapshots()` リスナーを使用し、新しい申請が届くとリストに即時追加される。手動リフレッシュ不要のシームレスな体験
- **拒否は静かに処理**: 承認時は成功ダイアログで祝福感を演出するが、拒否時はダイアログなしでカードが消えるだけ。ネガティブなアクションを目立たせない配慮
- **承認時のアニメーション**: カードのフェードアウト（duration: 300ms）で消失を自然に表現
- **プロフィール遷移なし**: この画面からは申請者のプロフィール画面に遷移しない。友達になった後にホーム画面の友達リストからアクセスする設計。情報は名前・ID・アバターの3要素で判断できる十分な量
- **申請者IDの取得**: `friend_requests` には `fromName` と `fromIconUrl` は非正規化で持っているが、`userId`（検索用ID）は持っていない。表示のために `fromUid` で `users` コレクションを参照する必要がある。パフォーマンス考慮で一括取得を推奨
