# アカウント設定画面 — Account Settings

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象画面**: アカウント設定画面

---

## 画面概要

メールアドレス変更・パスワード変更・ログアウト・アカウント削除を行う画面。
設定画面の「Account Settings」から遷移する。
各操作は破壊的・不可逆なものを含むため、段階的な確認フローを設ける。

## ダイアログ詳細

### メールアドレス変更ダイアログ

Firebase Authのメールアドレス変更には再認証が必要なため、現在のパスワード入力を求める。

| 要素 | 詳細 |
|------|------|
| タイトル | 「メールアドレスの変更」— ボールド、16px、`#FFFFFF` |
| 現在のメール表示 | 「現在: user@example.com」— 14px、`#A0A0A0` |
| 新しいメールアドレス入力 | `CustomTextField`。プレースホルダー「新しいメールアドレス」。`keyboardType: emailAddress` |
| パスワード入力 | `CustomTextField`。プレースホルダー「現在のパスワード」。`obscureText: true`、表示/非表示トグル |
| 「キャンセル」ボタン | `CustomButton`（secondary バリアント） |
| 「変更」ボタン | `CustomButton`（primary バリアント） |

- **実装時**: `CustomDialog` を使用
- **ボタン配置**: 横並び（左: キャンセル、右: 変更）、間隔 12px
- **バリデーション**: メールアドレス形式チェック、パスワード空欄チェック

**処理フロー:**
```
[変更ボタンタップ]
  ├── バリデーション
  │     ├── メール形式不正 → インラインエラー:「有効なメールアドレスを入力してください」
  │     └── パスワード空欄 → インラインエラー:「パスワードを入力してください」
  ├── ローディング表示（ボタン非活性化 + CustomLoadingIndicator）
  ├── Firebase Auth 再認証（reauthenticateWithCredential）
  │     └── 失敗 → エラーダイアログ:「パスワードが正しくありません」
  ├── Firebase Auth メールアドレス更新（verifyBeforeUpdateEmail）
  ├── Firestore users/{myUid}.email を更新
  ├── 成功 → 完了ダイアログ:「確認メールを送信しました。新しいメールアドレスを確認してください」
  └── 失敗 → エラーダイアログ（toUserFriendlyError で変換）
```

### アカウント削除フロー

不可逆操作のため、3ステップの慎重な確認フローを設ける。

#### ステップ1: 警告ダイアログ

| 要素 | 詳細 |
|------|------|
| アイコン | `warning_amber`、48px、`#FF4D4D` |
| タイトル | 「アカウントを削除」— ボールド、18px、`#FF4D4D` |
| 説明文 | 以下の影響を箇条書きで表示 — 14px、`#A0A0A0` |
| 「キャンセル」ボタン | `CustomButton`（secondary バリアント） |
| 「削除に進む」ボタン | `CustomButton`（danger バリアント） |

**説明文の内容:**
- 「全てのメッセージ・トーク履歴が削除されます」
- 「友達リストが全て解除されます」
- 「参加中のグループから退出されます」
- 「この操作は取り消せません」

#### ステップ2: パスワード再入力ダイアログ

ステップ1で「削除に進む」をタップ後に表示。

| 要素 | 詳細 |
|------|------|
| タイトル | 「本人確認」— ボールド、16px、`#FFFFFF` |
| 説明文 | 「アカウント削除を続けるには、パスワードを入力してください」— 14px、`#A0A0A0` |
| パスワード入力 | `CustomTextField`。プレースホルダー「パスワード」。`obscureText: true`、表示/非表示トグル |
| 「キャンセル」ボタン | `CustomButton`（secondary バリアント） |
| 「アカウントを削除」ボタン | `CustomButton`（danger バリアント） |

**処理フロー:**
```
[アカウントを削除ボタンタップ]
  ├── パスワード空欄チェック
  ├── ローディング表示（ボタン非活性化 + CustomLoadingIndicator）
  ├── Firebase Auth 再認証（reauthenticateWithCredential）
  │     └── 失敗 → エラーダイアログ:「パスワードが正しくありません」
  ├── Cloud Functions 呼び出し（アカウント削除処理）
  │     ├── users/{myUid} ドキュメント削除
  │     ├── 全友達の friends/{myUid} サブコレクションからドキュメント削除
  │     ├── 参加中チャットの members 配列から自身のUIDを除去
  │     ├── 関連する friend_requests を削除
  │     └── Firebase Auth ユーザー削除
  ├── 成功 → ログイン画面へ遷移（スタックを全てクリア）
  └── 失敗 → エラーダイアログ（toUserFriendlyError で変換）
```

## Firestoreとの対応

| UI要素 | Firestoreフィールド/操作 |
|--------|------------------------|
| 現在のメールアドレス表示 | `users/{myUid}.email`（読取） |
| メールアドレス変更 | Firebase Auth `verifyBeforeUpdateEmail` + `users/{myUid}.email` 更新 |
| パスワード変更 | Firebase Auth `updatePassword`（Firestoreへの書き込みなし） |
| ログアウト | Firebase Auth `signOut`（Firestoreへの書き込みなし） |
| アカウント削除 | Cloud Functions経由で `users/{myUid}` 削除 + 関連データクリーンアップ + Firebase Auth ユーザー削除 |

## エッジケース

| ケース | 対応 |
|--------|------|
| メールアドレス変更後に確認メール未承認 | Firebase Auth の `verifyBeforeUpdateEmail` を使用。確認完了までは旧メールアドレスが有効。完了ダイアログで確認メールの案内を表示 |
| 再認証トークン期限切れ | Firebase Auth が `requires-recent-login` エラーを返す → 自動的にパスワード入力を求める再認証フローへ |
| アカウント削除中にネットワーク切断 | Cloud Functions側で処理が途中の場合、リトライ機構で完了を保証。クライアントにはエラーダイアログを表示 |
| 新メールアドレスが既に登録済み | Firebase Auth が `email-already-in-use` エラーを返す → `toUserFriendlyError` で「このメールアドレスは既に使用されています」に変換 |
| パスワードが弱い | Firebase Auth が `weak-password` エラーを返す → `toUserFriendlyError` で変換 |
| ネットワークエラー | エラーダイアログ:「通信エラーが発生しました」 |

## デザインノート

- **設定画面との役割分担**: 設定画面にログアウトボタンがあるが、アカウント設定画面にも配置。設定画面のログアウトは「クイックアクセス」、本画面は「アカウント操作の一覧」としての位置づけ。導線が2箇所あっても混乱はない（どちらも同じ確認ダイアログを表示）
- **DANGER ZONEの分離**: ACCOUNTセクション（日常的な変更操作）とDANGER ZONEセクション（破壊的操作）を視覚的に分離。セクションラベルを赤色にすることで注意喚起
- **アカウント削除の3ステップ**: 不可逆操作のため、警告ダイアログ（影響の説明）→ パスワード再入力（本人確認）の2段階確認。誤操作のリスクを最小化
- **アイテムにアイコンなし**: 設定画面と同様、左側にアイコンを配置しないミニマルな構成。ただしDelete Accountのみテキストとアイコンを赤色にして視覚的に区別
- **ダイアログ内の入力**: メールアドレス・パスワード変更はダイアログ内で完結。専用画面への遷移なしで操作を簡潔に保つ
