# サイドドロワー — Side Drawer

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象コンポーネント**: サイドドロワー（Drawer）

---

## 画面概要

ホーム画面・トーク一覧画面の AppBar 左端ハンバーガーメニュー（`menu` アイコン）から開くサイドドロワー。ユーザープロフィールのサマリー、現在のプラン表示とアップグレード導線、ナビゲーションリンクを提供する。Emoff の「ストイックで無駄を排す」哲学に沿い、掲載項目は最小限に絞る。

## 開き方

- ホーム画面 AppBar の `menu` アイコンをタップ
- トーク一覧画面 AppBar の `menu` アイコンをタップ
- 左端からの右スワイプジェスチャー（Flutter の `Drawer` 標準動作）

## ドロワー構成（上から順）

### 1. ヘッダー — ユーザープロフィールサマリー

| 要素 | 詳細 |
|------|------|
| 背景 | `#1A1A1A` 全幅、パディング上48px（SafeArea考慮）・下24px・左右24px |
| アバター画像 | 64×64px、円形、ボーダー2px `#242424`。未設定時はイニシャルアイコン（`#242424` 背景、`#00D4FF` テキスト） |
| ユーザー名 | ボールド、18px、Manrope、`#FFFFFF` |
| ユーザーID | 14px、`#A0A0A0`、`@` プレフィックス付き |
| プランバッジ | ユーザー名の右横にインラインバッジ。詳細は下記 |

**プランバッジの表示:**

| プラン | バッジテキスト | スタイル |
|--------|-------------|---------|
| Free | `FREE` | `#242424` 背景、`#A0A0A0` テキスト、角丸8px、10px、ボールド |
| Personal Pro | `PRO` | `#00D4FF` 背景、`#0D0D0D` テキスト、角丸8px、10px、ボールド |
| Business | `BIZ` | `#7EEEFF` 背景、`#0D0D0D` テキスト、角丸8px、10px、ボールド |

**要素の配置:**
- アバターは左寄せ
- アバターの下に12px間隔でユーザー名行（名前 + プランバッジ、横並び、4px間隔）
- ユーザー名行の下に4px間隔でユーザーID
- ヘッダー下部に1pxの区切り線（`#242424`）

### 2. プランセクション（Free ユーザーのみ表示）

Free プランのユーザーにのみ表示するアップグレード促進エリア。Pro/Business ユーザーには非表示。

| 要素 | 詳細 |
|------|------|
| コンテナ | `#242424` 背景、角丸12px、パディング16px、マージン左右16px・上16px |
| アイコン | `rocket_launch`、24px、`#00D4FF` |
| タイトル | 「Upgrade to Pro」— セミボールド、14px、`#FFFFFF` |
| 説明テキスト | 「Unlock unlimited messages & group chats」— 12px、`#A0A0A0` |
| アップグレードボタン | 「View Plans」— カプセル型、`#00D4FF` 背景、`#0D0D0D` テキスト、12px、セミボールド、高さ32px |

**配置:**
- アイコンとタイトルは横並び（8px 間隔）
- 説明テキストはタイトルの下に4px間隔
- ボタンは説明テキストの下に12px間隔、左寄せ
- **タップ時**: サブスクリプション/課金画面（5-2）へ遷移

## インタラクション

| アクション | 動作 |
|------------|------|
| ハンバーガーメニューをタップ | ドロワーを開く |
| 右スワイプ（画面左端から） | ドロワーを開く |
| オーバーレイ部分をタップ | ドロワーを閉じる |
| 左スワイプ（ドロワー上で） | ドロワーを閉じる |
| アバターをタップ | プロフィール設定画面へ遷移（ドロワーを閉じてから遷移） |
| 「View Plans」をタップ | サブスクリプション/課金画面へ遷移（ドロワーを閉じてから遷移） |
| 「Subscription」をタップ | サブスクリプション/課金画面へ遷移（ドロワーを閉じてから遷移） |
| 「Announcements」をタップ | お知らせ画面へ遷移（ドロワーを閉じてから遷移） |
| 「Settings」をタップ | ドロワーを閉じ、ボトムナビの設定タブへ切り替え |

## デザインノート

- **掲載項目は3つに絞る**: 設定画面と重複する項目（通知設定、アカウント設定等）はドロワーに載せない。ドロワーはあくまで「よく使う近道」としての役割に限定
- **プランセクション**: Free ユーザーにのみ表示し、自然なアップグレード導線を提供。既にフェーズ3で設計したトークルームの Free 上限到達 UI と合わせて、2つの導線から課金画面へ誘導
- **Subscription アイテム**: Pro/Business ユーザーは現在のプラン管理のためにアクセス。Free ユーザーはプランセクションのボタンまたはこのアイテムからアクセス
- **Settings の遷移方法**: 別画面への push ではなく、ボトムナビの設定タブへの切り替え。ユーザーの位置感覚を維持し、「設定画面がどこにあるか分からない」問題を防ぐ
- **プロフィール情報の冗長性**: 設定画面にもプロフィールサマリーがあるが、ドロワーのそれは「今どのアカウントでログインしているか」の即時確認が目的。編集機能は持たず、タップでプロフィール設定画面に飛ばす
- **お知らせへの導線**: 設定画面内にもあるが、ドロワーにも配置することで到達ステップを削減（設定タブ → 設定画面 → お知らせ の3ステップが、ドロワー → お知らせ の2ステップに）
- **実装時の共通ウィジェット化**: `lib/widgets/custom_drawer.dart` として作成し、ホーム画面・トーク一覧画面の双方から利用する
