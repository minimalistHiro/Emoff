# パスワードリセット画面 — Reset Your Access.

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象画面**: パスワードリセット画面

---

## 画面概要

登録済みメールアドレスにパスワードリセットリンクを送信する画面。Firebase Authの `sendPasswordResetEmail` を使用する。入力項目はメールアドレスのみのシンプルな構成。

## 画面構成（上から順）

### 1. AppBar

| 要素 | 詳細 |
|------|------|
| 左アイコン | 戻るボタン（`arrow_back_ios`）、`#FFFFFF` |
| タイトル | なし（空） |
| 背景 | 透明（`#0D0D0D` と一体化） |

- **実装時**: `CustomAppBar` を使用
- **戻るボタンタップ**: ログイン画面へ `pop`

### 2. ヘッダー

| 要素 | 詳細 |
|------|------|
| メインタイトル | 「Reset Password」— ボールド、24px |
| サブテキスト | 「Enter your email address and we'll send you a link to reset your password.」— 通常ウェイト、14px |

- **配置**: 左寄せ
- **タイトル色**: `#FFFFFF`
- **サブテキスト色**: `#A0A0A0`
- **上マージン**: 24px（AppBarの下から）
- **下マージン**: 40px

### 3. メールアドレス入力フィールド

| 要素 | 詳細 |
|------|------|
| ラベル | 「EMAIL ADDRESS」— 大文字、11px、レタースペーシング広め、`#A0A0A0` |
| 入力フィールド | プレースホルダー「name@domain.com」 |
| キーボードタイプ | `emailAddress` |
| バリデーション | 必須、メール形式チェック |

- **実装時**: `CustomTextField` を使用
- **下マージン**: 32px

### 4. 送信ボタン

| 要素 | 詳細 |
|------|------|
| テキスト | 「Send Reset Link」+ 右矢印アイコン（`arrow_forward`） |
| 形状 | カプセル型（StadiumBorder）、横幅いっぱい |
| 高さ | 56px |

- **実装時**: `CustomButton`（primary バリアント）を使用
- **背景色**: `#00D4FF`（シアン）
- **テキスト色**: `#0D0D0D`（黒）

### 5. ログインへ戻るリンク

| 要素 | 詳細 |
|------|------|
| テキスト | 「Back to」— 14px、`#A0A0A0` |
| リンク | 「Login」— 14px、セミボールド、`#00D4FF`（シアン） |

- **配置**: 中央寄せ
- **上マージン**: 32px
- **タップ動作**: ログイン画面へ遷移（`pop`）

## 背景演出

| 要素 | 詳細 |
|------|------|
| メイン背景 | `#0D0D0D`（ダーク） |
| アンビエントグロー（任意） | ログイン画面・新規登録画面と同様 |

## 送信完了ダイアログ

送信成功時に表示するダイアログ。

| 要素 | 詳細 |
|------|------|
| タイトル | 「メール送信完了」— ボールド、18px、`#FFFFFF` |
| 本文 | 「パスワードリセット用のリンクを送信しました。メールをご確認ください。」— 14px、`#A0A0A0` |
| ボタン | 「ログイン画面に戻る」— `CustomButton`（primary）、横幅いっぱい |

- **実装時**: `CustomDialog` を使用
- **ボタンタップ時**: ダイアログを閉じてログイン画面へ遷移（`pop`）

## インタラクション

| アクション | 動作 |
|------------|------|
| 送信ボタンをタップ | バリデーション → `FirebaseAuth.instance.sendPasswordResetEmail(email)` → 成功時: 送信完了ダイアログ表示 |
| 送信完了ダイアログ「ログイン画面に戻る」 | ログイン画面へ遷移（`pop`） |
| AppBar 戻るボタン | ログイン画面へ遷移（`pop`） |
| 「Login」リンクをタップ | ログイン画面へ遷移（`pop`） |
| バリデーションエラー | `CustomDialog` でエラーメッセージ表示 |
| Firebase エラー（未登録メール等） | `toUserFriendlyError()` で変換した日本語メッセージを `CustomDialog` で表示 |

## デザインノート

- **シンプル構成**: メールアドレス1フィールド + 送信ボタンのみ。認証フロー3画面の中で最もミニマルな画面
- **ログイン画面との一貫性**: ラベルスタイル（大文字+広レタースペーシング）・ボタンスタイル・背景演出を統一
- **AppBar**: この画面はログイン画面からの遷移のため、戻るボタン付きのAppBarを配置。スプラッシュ・ログイン・新規登録画面にはAppBarがないが、パスワードリセット画面はサブ画面的位置づけのため追加
- **送信完了後のフロー**: ダイアログで完了を通知し、ユーザーが明示的に「ログイン画面に戻る」をタップして遷移する（自動遷移しない）
