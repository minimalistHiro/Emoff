# 新規登録画面 — Join the Philosophy.

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象画面**: 新規登録画面、AI処理同意ポップアップ、オンボーディング画面

---

## 画面概要

メールアドレスとパスワードで新規アカウントを作成する画面。登録完了後にAI処理同意ポップアップを表示し、同意後にオンボーディング画面（3スライド）を経てホーム画面へ遷移する。

**遷移フロー:**
```
新規登録画面 → [登録ボタン] → Firebase Auth アカウント作成
  → AI処理同意ポップアップ → [同意] → オンボーディング画面（3スライド）
  → [はじめる] → ホーム画面
```

## Part 2: AI処理同意ポップアップ

登録ボタンタップ → Firebase Authアカウント作成成功後に表示するダイアログ。

### ダイアログ構成

| 要素 | 詳細 |
|------|------|
| タイトル | 「AI処理について」— ボールド、18px、`#FFFFFF` |
| アイコン | なし（テキスト中心のシンプル構成） |

- **実装時**: `CustomDialog` を使用
- **背景色**: `#242424`
- **閉じる操作**: バリアタップ無効（`barrierDismissible: false`）。必ずボタンで操作させる

### 同意内容（スクロール領域）

以下4項目を箇条書きで表示。各項目の先頭に番号アイコン（シアン円 + 白数字）を配置。

| # | テキスト |
|---|---------|
| 1 | あなたが入力したテキストは、AI変換のため外部サービス（Anthropic）に送信されます |
| 2 | 変換前のテキストはサーバーに1ヶ月間保存された後、自動的に削除されます |
| 3 | Anthropic側では入出力データが最大7日間保持され、その後削除されます。モデルのトレーニングには使用されません |
| 4 | あなたはいつでもデータの削除を請求できます |

- **テキスト色**: `#FFFFFF`（主テキスト）、14px
- **番号アイコン**: 24px円、背景 `#00D4FF`、数字 `#0D0D0D`、12px、ボールド
- **項目間マージン**: 16px
- **ダイアログ内コンテンツが長い場合**: スクロール可能にする

### ボタン領域

| ボタン | 詳細 |
|--------|------|
| 「同意して始める」 | `CustomButton`（primary）、横幅いっぱい |
| 「同意しない」 | `CustomButton`（secondary）、横幅いっぱい |

- **「同意して始める」タップ時**: オンボーディング画面へ遷移
- **「同意しない」タップ時**: 作成したアカウントを削除し、新規登録画面に戻る。確認ダイアログ等は不要（まだ何もデータがないため）
- **ボタン間マージン**: 12px

## インタラクション（全体フロー）

| アクション | 動作 |
|------------|------|
| 登録ボタンをタップ | バリデーション → Firebase Auth でアカウント作成 → Firestore `users` ドキュメント作成（name, email, createdAt, updatedAt） → AI処理同意ポップアップ表示 |
| AI同意ポップアップ「同意して始める」 | オンボーディング画面へ遷移（`pushReplacement`） |
| AI同意ポップアップ「同意しない」 | 作成したアカウントを削除 → 新規登録画面に戻る |
| オンボーディング「はじめる」/「スキップ」 | ホーム画面へ遷移（`pushReplacement`） |
| 「Login」リンクをタップ | ログイン画面へ遷移 |
| 「利用規約」をタップ | 利用規約画面へ遷移（`push`） |
| 「プライバシーポリシー」をタップ | プライバシーポリシー画面へ遷移（`push`） |
| バリデーションエラー | `CustomDialog` でエラーメッセージ表示 |
| Firebase認証エラー | `toUserFriendlyError()` で変換した日本語メッセージを `CustomDialog` で表示 |

## デザインノート

- **ログイン画面との一貫性**: ブランディング領域・ラベルスタイル（大文字+広レタースペーシング）・ボタンスタイル・背景演出はログイン画面と統一
- **AI同意ポップアップ**: 法的要件を満たしつつ、読みやすさを重視。番号付き箇条書きで4項目を簡潔に提示
- **オンボーディング**: 3スライドに絞り、離脱リスクを最小化。スキップ可能にすることで急ぎたいユーザーにも対応
- **Firestoreドキュメント作成**: 登録時に `users/{uid}` に `name`, `email`, `createdAt`, `updatedAt` を書き込む。`userId`（友達検索用ID）は登録時には未設定で、プロフィール設定画面で後から設定する想定
