# 友達申請画面 — Find & Connect

- **作成日**: 2026-04-03
- **ステータス**: 設計済み・未実装
- **対象画面**: 友達申請画面

---

## 画面概要

ユーザーIDで他のユーザーを検索し、友達申請を送信する画面。
ホーム画面の `person_add` アイコンまたはFABから遷移する。
シンプルな「検索 → プレビュー → 申請」の3ステップフローで構成。

---

## 画面構成（上から順）

### 1. AppBar（固定ヘッダー）

| 要素 | 位置 | 詳細 |
|------|------|------|
| 戻るアイコン | 左端 | `arrow_back_ios` アイコン。ホーム画面に戻る |
| 画面タイトル「ADD FRIEND」 | 左寄せ（アイコン右） | 大文字・セミボールド・トラッキング広め |

- **実装時**: `CustomAppBar` を使用（UI_UX.mdルール準拠）
- **背景色**: `#0D0D0D`
- **アイコン色**: `#00D4FF`（シアン）

### 2. 説明セクション

| 要素 | 詳細 |
|------|------|
| メインテキスト | 「IDで友達を検索」— ボールド、20px、`#FFFFFF` |
| サブテキスト | 「相手のユーザーIDを入力して友達申請を送りましょう」— 通常ウェイト、14px、`#A0A0A0` |

- **配置**: 左寄せ、AppBar下に余白 24px
- **下マージン**: 32px

### 3. 検索入力エリア

| 要素 | 詳細 |
|------|------|
| IDプレフィックス | `@` マーク — 16px、`#A0A0A0`、入力フィールドの `prefixIcon` として配置 |
| テキスト入力フィールド | プレースホルダー:「ユーザーIDを入力」 |
| 検索ボタン | 入力フィールドの右に配置。`search` アイコン、48×48px、シアン背景（`#00D4FF`）、アイコン色 `#0D0D0D` |

- **実装時**: `CustomTextField` を使用
- **レイアウト**: `Row` で入力フィールド（`Expanded`）+ 検索ボタン（固定幅）を横並び
- **検索ボタン形状**: 円形（`CircleBorder`）
- **検索ボタン間隔**: 入力フィールドとの間隔 12px
- **入力制限**: 半角英数字・アンダースコアのみ（正規表現: `^[a-zA-Z0-9_]+$`）
- **下マージン**: 24px

### 4. 検索結果エリア（状態に応じて切り替え）

この領域は検索状態に応じて以下の4つの表示を切り替える。

#### 4-a. 初期状態（検索前）

| 要素 | 詳細 |
|------|------|
| イラスト/アイコン | `person_search` アイコン、64px、`#555555` |
| テキスト | 「IDを入力して検索してください」— 14px、`#555555`、中央寄せ |

- **配置**: エリア中央

#### 4-b. 検索中

| 要素 | 詳細 |
|------|------|
| ローディング | `CustomLoadingIndicator` を中央に表示 |
| テキスト | 「検索中...」— 14px、`#A0A0A0`、中央寄せ |

#### 4-c. ユーザー発見（メインフロー）

カード形式でユーザー情報を表示。

| 要素 | 詳細 |
|------|------|
| カード背景 | `#1A1A1A`、角丸 16px、パディング 20px |
| アバター画像 | 72×72px、円形。アイコン未設定時はデフォルトアバター（`person` アイコン、`#555555` 背景） |
| ユーザー名 | ボールド、18px、`#FFFFFF` |
| ユーザーID | `@userId` 形式、14px、`#A0A0A0` |
| 自己紹介文 | 最大2行（`maxLines: 2`, `overflow: ellipsis`）、14px、`#A0A0A0`。未設定時は非表示 |
| 申請ボタン | 「友達申請を送る」— `CustomButton`（primary バリアント）、カード下部に配置 |

- **カード配置**: 左右マージン 0px（画面パディングに準拠）
- **カード内レイアウト**: アバターを上部中央、テキスト情報を中央寄せで配置（縦並び）
- **申請ボタン**: カード内下部、上マージン 20px

**条件分岐による申請ボタンの変化:**

| 条件 | ボタン表示 | バリアント | 動作 |
|------|-----------|-----------|------|
| 通常（申請可能） | 「友達申請を送る」 | `primary` | 申請送信 |
| 既に友達 | 「友達です」 | `secondary`（非活性） | タップ不可 |
| 申請送信済み（pending） | 「申請済み」 | `secondary`（非活性） | タップ不可 |
| 相手から申請受信中 | 「申請が届いています」 | `primary` | 友達申請管理画面へ遷移 |
| 自分自身 | 「自分のIDです」 | `secondary`（非活性） | タップ不可 |

#### 4-d. ユーザー未発見

| 要素 | 詳細 |
|------|------|
| アイコン | `person_off` アイコン、64px、`#555555` |
| テキスト | 「ユーザーが見つかりませんでした」— 14px、`#A0A0A0`、中央寄せ |
| サブテキスト | 「IDを確認して再度検索してください」— 12px、`#555555`、中央寄せ |

### 5. 自分のID表示セクション

画面下部に自分のIDを表示し、共有を促す。

| 要素 | 詳細 |
|------|------|
| 区切り線 | 1px、`#242424`、上マージン 32px |
| ラベル | 「あなたのID」— 12px、大文字、`#A0A0A0`、レタースペーシング広め |
| 自分のID | `@myUserId` 形式、16px、ボールド、`#FFFFFF` |
| コピーアイコン | `content_copy`、20px、`#00D4FF`。タップでクリップボードにコピー |

- **配置**: 画面最下部（ボトムナビの上）に固定
- **レイアウト**: `Row` でID表示（`Expanded`）+ コピーアイコンを横並び
- **パディング**: 上下 16px、左右は画面パディングに準拠
- **コピー時**: `CustomDialog`（情報ダイアログ）で「IDをコピーしました」と表示

## Firestoreとの対応

| UI要素 | Firestoreクエリ/操作 |
|--------|---------------------|
| 検索フィールド入力 → 検索ボタン | `users` コレクションを `where('userId', '==', inputValue)` で検索 |
| 検索結果のアバター | `users.iconUrl` |
| 検索結果のユーザー名 | `users.name` |
| 検索結果のユーザーID | `users.userId` |
| 検索結果の自己紹介文 | `users.bio` |
| 「友達申請を送る」ボタン | `friend_requests` に新規ドキュメント作成: `{ fromUid: myUid, toUid: targetUid, fromName: myName, fromIconUrl: myIconUrl, status: 'pending', createdAt: now, updatedAt: now }` |
| 「既に友達」判定 | `users/{myUid}/friends/{targetUid}` の存在チェック |
| 「申請済み」判定 | `friend_requests` を `where('fromUid', '==', myUid).where('toUid', '==', targetUid).where('status', '==', 'pending')` で検索 |
| 「申請が届いています」判定 | `friend_requests` を `where('fromUid', '==', targetUid).where('toUid', '==', myUid).where('status', '==', 'pending')` で検索 |
| 自分のID表示 | 現在ログイン中のユーザーの `users.userId` |

## インタラクション

| アクション | 動作 |
|------------|------|
| 戻るアイコンをタップ | ホーム画面に戻る（`Navigator.pop`） |
| 検索ボタンをタップ | 入力されたIDで `users` コレクションを検索。結果に応じて4-a〜4-dを切り替え |
| キーボードの完了/Enterキー | 検索ボタンと同じ動作 |
| 「友達申請を送る」をタップ | 確認ダイアログ表示 →「〇〇さんに友達申請を送りますか？」→ 確認で申請送信 |
| 申請送信成功 | 成功ダイアログ表示 →「友達申請を送りました」→ ボタンが「申請済み」に変化 |
| 申請送信失敗 | エラーダイアログ表示（`toUserFriendlyError` で変換） |
| 「申請が届いています」をタップ | 友達申請管理画面へ遷移（`Navigator.push`） |
| コピーアイコンをタップ | 自分のIDをクリップボードにコピー + 情報ダイアログ表示 |
| 空文字で検索ボタンをタップ | 何も起きない（ボタン非活性化） |

## バリデーション

| チェック項目 | タイミング | エラー表示 |
|-------------|-----------|-----------|
| 空文字入力 | 検索ボタンタップ時 | 検索ボタンを非活性化（`#555555`、タップ不可） |
| 不正文字（記号・日本語等） | 入力時リアルタイム | `CustomTextField` のエラーテキスト:「半角英数字とアンダースコアのみ使用できます」 |
| 自分自身のID | 検索結果表示時 | カード表示 + ボタン「自分のIDです」（非活性） |
| ブロック中のユーザー | 検索結果表示時 | 4-d（ユーザー未発見）と同じ表示（ブロック情報は非公開） |

## エッジケース

| ケース | 対応 |
|--------|------|
| ネットワークエラー | エラーダイアログ:「通信エラーが発生しました。ネットワーク接続を確認してください」 |
| 自分をブロックしている相手を検索 | 通常通りユーザー情報を表示し申請可能（ブロック側の処理はCloud Functions側で制御） |
| 短時間に連続検索 | 前回の検索完了前に新しい検索を実行した場合、前回の結果を破棄 |
| 申請の重複送信 | `friend_requests` に既存の pending ドキュメントがある場合、「申請済み」表示で防止 |

## デザインノート

- **シンプルさ重視**: 検索→プレビュー→申請の3ステップに絞り、余計な要素を排除。LINEの「友だち追加」よりもミニマルなUI
- **自分のID表示**: 画面下部に常時表示することで「相手にIDを教える」フローもこの画面で完結させる。口頭やメッセージでIDを共有する際のアクセスポイント
- **ブロックユーザーの扱い**: 検索結果に表示しない（「見つかりませんでした」と同じ）。ブロックされている事実をユーザーに伝えないプライバシー配慮
- **状態遷移の明確さ**: 申請ボタンの表示を5パターンに分岐させることで、現在の関係性をボタン1つで伝える
- **レイアウト**: 検索結果カードは縦並びの中央寄せレイアウトで、プロフィール情報を見やすく配置
