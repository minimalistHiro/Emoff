## 概要

トークルーム画面の `more_vert` メニューからチャット相手をブロックするためのUI。友達プロフィール画面のブロック確認ダイアログ（2-3a）と同一仕様で、成功時の遷移先のみ異なる。

**対応Firestoreデータ:**
- `users/{uid}/blocked_users` サブコレクション

---

## 導線

トークルーム AppBar の `more_vert` メニューに「ブロック」項目を追加。

### メニュー構成（更新後）

| メニュー項目 | アイコン | テキスト色 | 表示条件 |
|-------------|---------|-----------|---------|
| グループ管理 | `group` | `#FFFFFF` | グループチャットのみ |
| 通報する | `flag` | `#FF4D4D` | 1対1チャットのみ（3-a設計済み） |
| ブロック | `block` | `#FF4D4D` | 1対1チャットのみ |

- **メニュースタイル**: 背景 `#242424`、角丸12px
- **区切り線**: 「通報する」と「ブロック」の間に1pxの区切り線（`#333333`）
- **グループチャットの場合**: 「通報する」「ブロック」は非表示（メッセージ長押しの通報、グループ管理のみ）

## ブロック確認ダイアログ

友達プロフィール画面（2-3a）と**同一仕様**。

| 要素 | 詳細 |
|------|------|
| アイコン | `block`、48px、`#FF4D4D` |
| タイトル | 「〇〇さんをブロック」— ボールド、18px、`#FFFFFF` |
| 説明文 | 以下の3点を箇条書き、14px、`#A0A0A0` |
| キャンセルボタン | 「キャンセル」— `CustomButton`（`secondary`バリアント） |
| ブロックボタン | 「ブロックする」— `CustomButton`（`danger`バリアント） |

**説明文の内容:**
- 「相手からのメッセージが届かなくなります」
- 「友達リストからお互いに削除されます」
- 「ブロックしたことは相手に通知されません」

- **実装時**: `CustomDialog` を使用
- **背景色**: `#242424`
- **角丸**: 16px
- **ボタン配置**: 横並び（左: キャンセル、右: ブロック）、ボタン間隔 12px

## ブロック処理フロー

```
[ブロックボタンタップ]
  ├── ローディング表示（ダイアログ内ボタンを非活性化 + CustomLoadingIndicator）
  ├── 処理実行:
  │     ① users/{myUid}/blocked_users/{targetUid} にドキュメント作成
  │     ② Cloud Functionsが以下を自動処理:
  │          - 両者の friends サブコレクションから相互削除
  │          - （チャットは削除しない。メッセージ非表示はクライアント側で制御）
  ├── 成功 → ダイアログ閉じる → **トーク一覧画面に戻る**（pushAndRemoveUntilでスタッククリア）
  └── 失敗 → エラーダイアログ表示（toUserFriendlyError で変換）
```

### 友達プロフィール画面（2-3a）との差異

| 項目 | 友達プロフィール画面（2-3a） | トークルーム画面（3-d） |
|------|--------------------------|----------------------|
| 導線 | AppBarメニュー → ブロック | AppBarメニュー → ブロック |
| ダイアログUI | 同一 | 同一 |
| 処理フロー | 同一 | 同一 |
| **成功時の遷移先** | **ホーム画面** | **トーク一覧画面** |

- トークルームからブロックした場合、トーク一覧に戻る。ブロックした相手とのチャットはトーク一覧から非表示になる（クライアント側でフィルタ）

## 実装時の共通化

ブロック確認ダイアログのUIと処理ロジックは、友達プロフィール画面（2-3a）と完全に同一。**共通ウィジェット/関数として実装し、遷移先のみパラメータで切り替える**ことを推奨。

```
BlockConfirmDialog(
  targetUserName: "〇〇",
  targetUid: "xxx",
  onSuccess: () => navigateToTalkList(),  // or navigateToHome()
)
```

---

## デザインノート

- **2-3aとの一貫性**: ブロック操作はアプリ内のどの導線からでも同じUI・同じ説明文で統一。ユーザーが操作の結果を常に同じように理解できる
- **1対1チャット限定**: グループチャットではブロックメニューを表示しない。グループ内の特定メンバーブロックは複雑になるため、MVP時点ではスコープ外
- **成功時の遷移先の違い**: 友達プロフィールからはホーム画面へ、トークルームからはトーク一覧へ。それぞれの画面スタックの文脈に沿った自然な遷移
- **共通ウィジェット化**: 同一UIを2箇所で使うため、コンポーネントとして切り出し、遷移先のコールバックだけを差し替える設計
