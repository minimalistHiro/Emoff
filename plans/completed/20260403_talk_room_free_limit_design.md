- **作成日**: 2026-04-03

- **対象画面**: トークルーム画面（既存設計への追加）

## 概要

Freeプランユーザーが1日30通のメッセージ変換上限に到達した際のUI。段階的に通知し、上限到達後は入力を制限してアップグレードを促す。

**ビジネスモデル参照:**
- Freeプラン: 1日30通まで変換可能、1対1チャットのみ
- Personal Pro（¥300/月）: 変換無制限、グループチャット、変換トーン選択

---

## 残数カウンター（常時表示）

Freeプランユーザーにのみ、インプットドック上部に残数バーを表示。

| 要素 | 詳細 |
|------|------|
| 残数テキスト | 「残り {N} 通」— 12px、`#A0A0A0` |
| プログレスバー | 横幅いっぱい、高さ 2px、角丸フル |
| バー背景色 | `#242424` |
| バー進捗色（通常） | `#00D4FF`（シアン） |
| バー進捗色（残り5通以下） | `#FFB800`（警告色） |
| バー進捗色（残り0通） | `#FF4D4D`（エラー色） |

- **配置**: インプットドックの直上、パディング左右16px、上8px、下4px
- **Pro以上のユーザー**: 非表示
- **日次リセット**: 毎日0:00（JST）にカウントリセット

## 段階的通知

### ステージ1: 残り5通（警告）

25通目の送信完了時に、メッセージキャンバス内にインラインバナーを表示。

| 要素 | 詳細 |
|------|------|
| アイコン | `info`、20px、`#FFB800` |
| テキスト | 「本日の残り送信回数はあと5通です」— 13px、`#FFB800` |
| 背景色 | `#FFB800` の8%不透明度 |
| 角丸 | 12px |
| 閉じるボタン | `close`、16px、`#A0A0A0`。右端 |

- **配置**: メッセージキャンバス内、最新メッセージの下にインライン表示
- **表示アニメーション**: フェードイン + スライドアップ（300ms）
- **閉じるタップ時**: バナーをフェードアウトで非表示

### ステージ2: 上限到達（30通送信完了）

30通目の送信完了時に、上限到達ダイアログを表示。

| 要素 | 詳細 |
|------|------|
| アイコン | `block`、48px、`#FF4D4D` |
| タイトル | 「本日の送信上限に達しました」— ボールド、18px、`#FFFFFF` |
| 説明テキスト | 「Freeプランでは1日30通までメッセージを送信できます。明日0:00にリセットされます。」— 14px、`#A0A0A0` |
| アップグレードボタン | 「Proプランにアップグレード」— `CustomButton`（`primary`バリアント） |
| 閉じるボタン | 「明日まで待つ」— `CustomButton`（`secondary`バリアント） |

- **実装**: `CustomDialog` を使用
- **「Proプランにアップグレード」タップ時**: サブスクリプション/課金画面へ遷移（未設計。暫定的にダイアログを閉じる）

## 上限到達後のインプットドック

30通到達後、インプットドックの表示を切り替える。

### 通常のインプットドック → ロック状態に変更

| 要素 | 詳細 |
|------|------|
| テキスト入力フィールド | 非活性化。プレースホルダー「本日の送信上限に達しました」、`#555555` |
| 「CONFIRM」ボタン | 非活性化。背景 `#333333`、テキスト `#555555`、タップ不可 |
| アップグレードリンク | インプットドック下部に「Proプランで無制限に →」テキストリンク、13px、`#00D4FF`、下線付き |

- **タップ時（アップグレードリンク）**: サブスクリプション/課金画面へ遷移
- **メッセージ受信**: 上限到達後も相手からのメッセージは**通常通り受信・表示される**。送信のみが制限される

## メッセージ送信時のカウント管理

```
送信フロー（既存の「SEND NEUTRALIZED」タップ後に追加）:
1. 送信処理の前にローカルのカウントを確認
2. カウント < 30 → 通常送信。カウント+1
3. カウント == 25到達 → 送信後にステージ1バナー表示
4. カウント == 30到達 → 送信後にステージ2ダイアログ表示 + インプットドックをロック状態に変更
5. カウント >= 30 → 送信ブロック（CONFIRMボタンが非活性のため到達しない）
```

**カウントの保存先:**
- ローカル（SharedPreferences）にカウントと日付を保存
- アプリ再起動時: 保存日付が今日でなければカウントを0にリセット
- サーバーサイド検証は将来的にCloud Functionsで追加予定（クライアント改ざん対策）

## デザインノート

- **段階的な通知**: いきなりブロックするのではなく、残り5通の警告→上限到達ダイアログ→インプットドックロックの3段階で体験を緩やかに制限
- **受信は制限しない**: 上限到達後も相手からのメッセージは受信できる。コミュニケーションが完全に途切れることを防ぐ
- **常時残数表示**: プログレスバーで現在の消費量を可視化し、ユーザーが計画的に使える
- **アップグレード導線の複数配置**: ダイアログのボタン + インプットドック下部のリンクで、異なるタイミングでの転換を狙う
- **色の段階変化**: シアン（通常）→ イエロー（警告）→ レッド（上限）のグラデーションで直感的に状態を把握
