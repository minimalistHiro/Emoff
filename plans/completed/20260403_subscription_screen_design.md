## 画面概要

プラン一覧の表示・比較・購入・管理を行う画面。Free ユーザーにはアップグレード訴求、Pro/Business ユーザーには現在のプラン管理を提供する。「Choose Your Clarity（明晰さを選ぶ）」をコンセプトに、シンプルで迷わないプラン選択体験を実現する。

**遷移元（4箇所）:**
1. サイドドロワー — 「View Plans」ボタン（Free ユーザー向けプランセクション）
2. サイドドロワー — 「Subscription」ナビアイテム（全ユーザー）
3. トークルーム — Free 上限到達ダイアログ「Proプランにアップグレード」ボタン
4. グループ作成画面 — Free ユーザー向けアップグレードダイアログ「プランを見る」ボタン

**ビジネスモデル参照:**
- Free（¥0）: 1日30通まで、1対1チャットのみ
- Personal Pro（¥300/月）: 変換無制限、グループチャット、変換トーン選択
- Business（¥600/ユーザー/月）: チーム管理、利用統計、グループチャット、優先サポート

**MVP方針:** Free + Personal Pro のみで開始。Business プランは「Coming Soon」表示とし、成長段階で有効化する。

## 購入フロー

### Free → Personal Pro アップグレード

```
1. 「Upgrade to Pro」ボタンをタップ
2. OS のアプリ内課金シート（StoreKit 2 / Google Play Billing）が表示される
3a. 購入成功 → 成功ダイアログ表示 → プラン状態を更新 → 画面を再描画
3b. 購入キャンセル → 何もしない（元の画面に戻る）
3c. 購入失敗 → エラーダイアログ表示
```

## プラン状態の管理

| 項目 | 詳細 |
|------|------|
| 購入方式 | App Store / Google Play のアプリ内サブスクリプション（自動更新型） |
| Flutter パッケージ | `in_app_purchase`（公式）または `purchases_flutter`（RevenueCat）を使用 |
| プラン判定 | アプリ起動時にサブスクリプション状態を検証。有効なサブスクリプションがなければ Free |
| サーバーサイド検証 | Cloud Functions でレシート検証を行い、`users/{uid}` に `plan` フィールド（`free` / `pro` / `business`）を書き込む |
| ローカルキャッシュ | SharedPreferences にプラン状態をキャッシュし、毎回のサーバー通信を回避 |

**Firestore `users` コレクションへの追加フィールド（実装時に FIRESTORE.md を更新）:**

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `plan` | string | `free` / `pro` / `business` |
| `planExpiresAt` | timestamp | サブスクリプション有効期限（サーバーサイドで管理） |

## インタラクション

| アクション | 動作 |
|------------|------|
| 戻るボタンをタップ | 遷移元に戻る |
| 「Upgrade to Pro」をタップ | OS のアプリ内課金フローを開始 |
| 「Current Plan」（非活性）をタップ | 何もしない |
| 「Manage Subscription」をタップ | OS のサブスクリプション管理画面を開く |
| 「Restore Purchases」をタップ | 購入履歴を復元。成功時は成功ダイアログ、該当なしの場合はその旨ダイアログ |
| 「利用規約」をタップ | 利用規約画面へ遷移 |
| 「プライバシーポリシー」をタップ | プライバシーポリシー画面へ遷移 |
| Business カード全体 | タップ不可（opacity でグレーアウト） |

## デザインノート

- **カード縦並び**: 横並びタブ切り替え方式ではなく、3枚を縦に並べて一覧比較できる構成。スクロールで全プランが見え、比較しやすい
- **Pro カードの強調**: シアンボーダー2px + RECOMMENDED バッジで視覚的に誘導。料金もシアンで統一し、他のカードとの差別化を図る
- **Business の Coming Soon**: MVP では選択不可だが、将来の拡張性を示す。opacity 60% で「準備中」を表現
- **打ち消し線の活用**: Free カードで使えない機能に打ち消し線を入れ、Pro との差を直感的に伝える
- **注釈セクション**: App Store / Google Play のレビューガイドラインで必須とされるサブスクリプション説明（自動更新・キャンセル方法・利用規約リンク）を配置
- **Restore Purchases**: App Store レビューガイドラインで必須。デバイス移行時やアプリ再インストール時の購入復元に対応
- **プラン状態のサーバーサイド管理**: クライアントのみでの判定はセキュリティリスクがあるため、Cloud Functions でのレシート検証を前提とする。ただし UX のためローカルキャッシュも併用
- **導線の統一**: 4箇所の遷移元からこの1画面に集約し、プラン情報の一元管理を実現。遷移元によって表示を変えない（全ユーザー同じ画面を見る）
